1NanoClawとは

NanoClawはTypeScript / Node.jsで構築された軽量AIエージェントフレームワークです。AnthropicのClaude Agent SDKを基盤とし、セッションごとにOSレベルのコンテナ分離を行うことで、高いセキュリティと柔軟性を実現しています。

開発元はqwibitai（GitHub: qwibitai/nanoclaw）。WhatsApp・Telegram・Slack・Discord・Gmailといった主要メッセージングプラットフォームと接続でき、複数の専門エージェントが協調する「Agent Swarm」機能も備えています。

NanoClawが選ばれる3つの理由

🔒

コンテナ分離

セッションごとにDocker / Apple Containerで完全隔離。悪意あるコードの横断実行を防止

🤖

Agent Swarm

役割を持った複数エージェントが協調し、複雑なタスクを並列処理

💬

マルチプラットフォーム

WhatsApp・Telegram・Slack・Discord・Gmailを単一インスタンスで統合

🛠️

TypeScript製

型安全・拡張性が高く、Claude Agent SDKと直接統合。カスタマイズも容易

💡 PicoClawとの棲み分け： 超低消費電力のハードウェア（RISC-V等）や<10MB RAM環境にはPicoClawを、セキュリティ重視・複数チャットアプリ統合・Agent Swarmが必要な用途にはNanoClawを選びましょう。

3セキュリティアーキテクチャ

NanoClawの最大の特徴はセッションごとのOSレベルコンテナ分離です。各会話セッションは独立したコンテナ内で実行され、他のセッションやホストシステムへのアクセスが制限されます。

コンテナ分離の仕組み

ユーザーメッセージ (WhatsApp / Telegram / Slack)
↓
NanoClaw ルーター（ホスト）
┌──────┴───────┐
↓ ↓
セッション A セッション B
┌─────────────┐ ┌─────────────┐
│ Docker/Apple│ │ Docker/Apple│
│ Container │ │ Container │
│ (分離環境) │ │ (分離環境) │
└─────────────┘ └─────────────┘

セキュリティ機能一覧

🏗️

プロセス隔離

各セッションが独立プロセス空間で動作。クロスセッション汚染なし

📁

ファイルシステム分離

コンテナごとに独立したファイルシステム。ホストへの直接書き込みを防止

🌐

ネットワーク制御

コンテナのネットワークアクセスを明示的なルールで制限可能

🗑️

自動クリーンアップ

セッション終了時にコンテナを自動削除。残留データなし

Apple Container（macOS）： macOS 15+では、DockerのかわりにApple純正のApple Containerフレームワークを利用できます。より軽量で、macOSとの統合が深く、起動が高速です。

4システム要件

対応OS

macOS 13+ (Ventura以降) Ubuntu 22.04 LTS Debian 12 Fedora 38+ Windows WSL2

必須ソフトウェア

  • Node.js 20.x 以上（LTS推奨）
  • npm 9.x 以上
  • Claude Code（Anthropic CLI）
  • Git 2.30 以上

  • Docker Desktop 24+ / Docker Engine
  • Apple Container（macOS 15+）
  • Podman 4.x（Linux）

ハードウェア最小要件

  • RAM: 2GB以上（推奨4GB+）
  • ストレージ: 5GB以上
  • CPU: x86_64 / ARM64

Anthropic認証情報

  • Anthropic API Key、または
  • Claude Code サブスクリプション
  • （どちらか一方でOK）
⚠️ コンテナエンジンは必須です。 DockerまたはApple Containerが起動していない場合、NanoClawのセッション分離機能は動作しません。事前に docker info で稼働確認をしてください。

5インストール手順

Step 1: 前提ソフトウェアのインストール

Node.js 20 LTS のインストール（Ubuntu / Debian）

# NodeSourceリポジトリを追加してNode.js 20をインストール
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs
node --version   # v20.x.x と表示されること

Node.js 20 LTS のインストール（macOS）

# Homebrew経由でインストール
brew install node@20
brew link node@20 --force
node --version

Claude Code のインストール

# npm経由でClaude Codeをグローバルインストール
npm install -g @anthropic-ai/claude-code
claude --version   # バージョン番号が表示されること

Docker のインストール（Linux）

# Docker公式スクリプトを使用
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER
newgrp docker
docker info   # Dockerが動作していることを確認

Step 2: NanoClawのクローン

# リポジトリをクローン
git clone https://github.com/qwibitai/nanoclaw.git
cd nanoclaw

# 依存パッケージをインストール
npm install

Step 3: Claude Codeのセットアップ起動

# プロジェクトルートでClaude Codeを起動
claude

# Claude Code内でNanoClawセットアップコマンドを実行
> /setup

# 対話形式でAPIキーや使用するプラットフォームを設定します

7Agent Swarm — 複数エージェント協調

NanoClaw最大のユニーク機能がAgent Swarmです。複数の専門エージェントがチームを形成し、複雑なタスクを分散処理します。

Swarmのアーキテクチャ

ユーザーリクエスト：「競合分析レポートを作成して」
↓
🦀 オーケストレーター（NanoClaw）
┌──────────┼──────────┐
↓ ↓ ↓
🔍 検索Agent 📊 分析Agent 📝 執筆Agent
Web検索・情報 データ整理・ レポート
収集 比較 執筆・整形
└──────────┼──────────┘
↓
統合された最終レポート → ユーザーへ送信

Swarmの設定方法

設定ファイルは存在せず、claude コマンドプロンプトでエージェントの役割を定義します。

# claude プロンプトでSwarm設定を追加
> /swarm create research-team

Define agent roles for "research-team" swarm:
? Agent 1 role: searcher — Web search and data gathering
? Agent 2 role: analyst  — Data processing and comparison
? Agent 3 role: writer   — Report generation and formatting
? Orchestration strategy: sequential / parallel: parallel

✓ Swarm "research-team" created with 3 agents
💡 Swarmのベストプラクティス： エージェント数は3〜5が最適です。多すぎると調整コストが上がり、レスポンスが遅くなります。タスクの性質に応じて sequential（順次）か parallel（並列）を選択してください。

ニュース収集・要約

収集Agent→要約Agent→送信Agentが連携し、カスタムニュースレターを自動生成

🛒

価格比較・購買支援

複数ECサイトを並列検索し、最安値・レビュー比較をまとめて報告

📊

データ分析パイプライン

データ取得・クレンジング・分析・可視化を各Agentが担当して自動化

10Tips & ベストプラクティス

💡 TIP 01

Docker BuildKitを有効化： DOCKER_BUILDKIT=1 環境変数を設定するとコンテナビルドが最大3倍高速化されます。

💡 TIP 02

コンテナイメージをキャッシュ： 初回起動後はイメージがキャッシュされます。docker image prune を定期実行してディスクを管理しましょう。

💡 TIP 03

セッションタイムアウトを設定： 長期間応答のないセッションのコンテナは自動終了させることでリソースを節約できます。claudeプロンプトで /config timeout 30m と設定します。

💡 TIP 04

Swarmのログを分けて管理： 各エージェントのログは npm run logs -- --agent searcher のようにフィルタリングできます。

💡 TIP 05

ヘルスチェックを活用： npm run health でNanoClaw全コンポーネントの稼働状態を一括確認できます。本番環境ではcronで定期実行推奨。

💡 TIP 06

APIレート制限に注意： Anthropic APIには1分あたりのリクエスト制限があります。高負荷時は /config rate-limit 30 で同時処理数を制限します。

💡 TIP 07

claude プロンプトでデバッグ： /debug session [ID] で特定セッションのコンテナ内ログをリアルタイム確認できます。

💡 TIP 08

WhatsApp本番環境はngrokを使わない： 開発時はngrokが便利ですが、本番では固定IPまたはドメインを持つサーバーに正式デプロイしましょう。

💡 TIP 09

Podmanを代替として使用： Dockerの代わりにRootless Podmanを使うとさらにセキュリティが向上します。/config container-engine podman で切り替え可能。

💡 TIP 10

Claude Code Subscriptionを優先： Anthropic API KeyよりClaude Code Subscriptionを使う方が、レート制限が緩く、高負荷環境に適しています。

11よくある質問（FAQ）

Q. PicoClawとどちらを選べばいいですか？
ハードウェアリソースが極端に限られている（RAM <256MB、RISC-VやARMマイコン）場合はPicoClaw、セキュリティ・複数チャットアプリ統合・Agent Swarmが必要な場合はNanoClawを選択してください。クラウドサーバー上での運用ならNanoClawが適しています。
Q. DockerなしでNanoClawを動かせますか？
コンテナ分離機能はDockerまたはApple Container（もしくはPodman）が必須です。ただし、セキュリティリスクを理解した上で開発・テスト環境のみに限定するなら、コンテナなし動作モードも存在しますが、本番環境では非推奨です。
Q. 設定ファイルはどこにありますか？
NanoClawはTOMLやYAMLなどの設定ファイルを使用しません。すべての設定はClaudeプロンプト（/config コマンド）か、TypeScriptコードとして管理されます。これにより型安全性と柔軟性が確保されます。
Q. コンテナはどのくらいのリソースを使いますか？
1セッションあたり約80〜150MBのRAMが必要です。同時接続ユーザーが多い場合はメモリに注意してください。docker stats コマンドでリアルタイムのリソース使用量を確認できます。
Q. Agent Swarmのエージェント数に制限はありますか？
技術的な上限はありませんが、実用的には3〜7エージェントが推奨です。エージェント数が増えると調整コスト（オーケストレーションのAPI呼び出し）も増加し、応答時間が長くなります。
Q. WhatsApp BusinessとWhatsApp個人版の違いは？
NanoClawが利用するWhatsApp Business APIは法人向けのものです。個人のWhatsAppアカウントには直接接続できません。Meta for Developersで申請が必要で、小規模なら無料枠内で利用可能です。
Q. macOSでApple ContainerとDockerのどちらが良いですか？
macOS 15（Sequoia）以上であればApple Containerを推奨します。Apple Siliconと深く統合されており、起動速度・メモリ効率ともにDockerより優れています。macOS 14以下はDockerを使用してください。
Q. 本番環境でのセキュリティチェックリストは？
①Dockerをrootlessモードで実行する、②APIキーを環境変数ではなくシークレット管理ツール（HashiCorp Vault等）で管理する、③Webhookエンドポイントに署名検証を実装する、④コンテナネットワークを最小権限で設定する、⑤定期的なコンテナイメージの更新を行う、の5点が重要です。
