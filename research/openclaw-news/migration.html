🔄 OpenClaw 移行ガイド

他のAIエージェントからOpenClawへのスムーズな移行手順

🏠 トップ 🔧 セットアップ 🏗️ アーキテクチャ 🔧 トラブルシューティング

1. AutoGPTからの移行

AutoGPTユーザーが最も多い移行パスです。基本的なワークフローの考え方は似ていますが、OpenClawのSkillsシステムがAutoGPTのプラグインシステムに相当します。

1

設定ファイルのエクスポート

AutoGPTの.envファイルからAPIキーと設定を抽出します。

# AutoGPTの設定をバックアップ
cp .env autogpt_backup.env

# OpenClawの設定ディレクトリを作成
mkdir -p ~/.openclaw/config
2

プラグイン→Skillへの変換

AutoGPTのプラグインをOpenClawのSkill形式に変換します。

# Skill変換ツールを使用
openclaw migrate --from autogpt --plugins-dir ./plugins

# 手動変換の場合：SKILL.mdテンプレートを作成
openclaw skill init my-converted-skill
3

ワークフローの移植

  • AutoGPTの「ゴール」→ OpenClawの「タスク定義」に対応
  • AutoGPTの「メモリ」→ OpenClawの「コンテキスト管理」に対応
  • AutoGPTの「制約」→ OpenClawの「権限設定」に対応

2. LangChainベースアプリからの移行

LangChainで構築したカスタムエージェントからOpenClawへ移行するケースです。

🔑 主な違い
  • チェーン構造： LangChainのチェーン/パイプラインはOpenClawのSkillシーケンスに相当
  • ツール定義： LangChainのToolクラスはSKILL.mdの宣言的記述に置き換わる
  • メモリ管理： LangChainのMemoryクラスはOpenClawのビルトインコンテキスト管理が担当
  • プロンプト管理： PromptTemplateはOpenClawのSkillメタデータに統合される

3. CrewAIからの移行

CrewAIのマルチエージェント構造からOpenClawへの移行では、エージェント定義をSkillとして再構築します。

  • Agent定義： CrewAIの各AgentはOpenClawの専門Skillに対応させる
  • Task定義： CrewAIのTaskはOpenClawのタスクキューに移植
  • Process管理： CrewAIのsequential/hierarchicalプロセスはOpenClawのワークフロー定義で再現
  • コールバック： CrewAIのコールバック機能はOpenClawのWebhookシステムに対応

4. データ移行チェックリスト

📋 移行前の確認事項
  • APIキー： 全てのLLMプロバイダーのAPIキーを準備（Claude, OpenAI, DeepSeek等）
  • 既存データ： 会話履歴、タスクログ、設定ファイルのバックアップ
  • 外部連携： メール、カレンダー、Slack等の連携設定を記録
  • カスタムロジック： 独自に実装した処理フローの文書化

5. バージョン間の移行

OpenClaw自体のメジャーバージョンアップグレード手順です。

⚠️ v0.x → v1.x 移行時の注意点
  • Skill形式が変更されているため、全Skillの更新が必要
  • 設定ファイルのスキーマが大幅に変更 — openclaw migrate-configコマンドを使用
  • 一部のバンドルSkillが削除・統合されているため、依存関係の確認が必要
  • 認証方式が平文→暗号化に変更。既存の認証情報の再設定が必要

6. 移行後の検証

  • 機能テスト： 全Skillが正常に動作することを確認
  • パフォーマンステスト： 移行前と同等以上のレスポンス速度を確認
  • セキュリティ監査： 認証情報が安全に保存されていることを確認
  • ログ確認： エラーや警告がないことを確認
