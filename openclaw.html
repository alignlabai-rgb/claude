OpenClawの役割

AIツール群の
指揮者として機能

OpenClawはワークフロー定義に従い、複数のAIツール・APIを順序立てて呼び出す オーケストレーションレイヤーとして機能します。 各ツールのAPI差異を吸収し、エラー処理・リトライ・進捗管理を一元化。

  • ワークフローのYAML/JSON定義で管理
  • 並列・直列の実行フロー制御
  • 各ステップの成功/失敗に応じた分岐処理
  • 実行ログ・コスト追跡・アラート機能
  • ヒューマンインザループ（人間確認ポイント）の挿入

ユーザーインターフェース

Signalメッセージで
全パイプラインをトリガー

ユーザーはSignalアプリからテキストを送るだけで全自動制作が始まります。 OpenClawがSignal Botとして常駐し、コマンドを解析してワークフローを起動。 進捗・完了・エラーを全てSignalで返信します。

  • /create video ネタ文章 → 動画生成を起動
  • /create manga ネタ文章 → 漫画生成を起動
  • /create both ネタ文章 → 動画+漫画を並列生成
  • /status 全ジョブの進捗を確認
  • /revenue today 当日の収益サマリーを取得
# Signal Bot ハンドラー (OpenClaw Skill) name: signal-bot-handler trigger: type: signal_message allowed_senders: - "${{env.OWNER_SIGNAL_NUMBER}}" on_message: - match: "/create both (.+)" workflow: dual-content-creation input_map: topic_text: "$1" reply: "動画+漫画の並列生成を開始しました 🚀" - match: "/status" workflow: job-status-report reply_template: "実行中: {{running}} | 完了: {{done}} | エラー: {{errors}}" - match: "/revenue today" workflow: revenue-summary reply_template: "本日収益: ¥{{total_jpy}} ({{platform_breakdown}})"
Skill登録・管理

各モジュールを
OpenClaw Skillとして登録

AI Creator Hubの各モジュールはOpenClaw Skillとして定義されます。 SKILL.mdファイル1つで入出力・エラー処理・コスト上限を宣言的に管理。 新しいモジュール追加はファイルを置くだけで自動認識されます。

# skills/ ディレクトリ構成 skills/ ├── video-pipeline/ │ ├── SKILL.md # Skill定義 │ ├── main.py # エントリーポイント │ └── requirements.txt ├── manga-pipeline/ │ ├── SKILL.md │ └── main.py ├── content-evaluator/ │ ├── SKILL.md │ └── evaluator.py ├── monetization-manager/ │ ├── SKILL.md │ └── publisher.py └── signal-bot-handler/ ├── SKILL.md └── bot.py

SKILL.md 標準フォーマット

# SKILL.md — 全Skillの共通フォーマット name: video-pipeline description: "ネタ文章から動画を自動生成" version: "2.1.0" author: "AI Creator Hub" cost_limits: per_run_usd: 2.00 # 1回あたり上限 per_day_usd: 30.00 # 日次上限 on_exceed: abort_notify # 超過時: 中断+通知 retry_policy: max_attempts: 3 backoff: exponential # 2s→4s→8s on_final_failure: human_alert timeout_sec: 5400 # 90分タイムアウト input: # ... (各Skill固有) output: # ... (各Skill固有)
ワークフロー定義

動画+漫画 デュアル
並列ワークフロー例

同じネタから動画と漫画を並列生成するメインワークフロー。共通前処理を1回だけ実行してコストを削減。

# dual_content_workflow.yaml name: "dual-content-creation" version: "2.0" trigger: type: "signal_command" command: "/create both" steps: - id: "analyze" skill: "llm.claude" # 共通前処理 prompt: "ジャンル・ターゲット・トーンを分析: {{input.topic_text}}" output: "topic_data" - id: "gen_video" # 並列A skill: "video-pipeline" parallel_group: "content_gen" depends_on: ["analyze"] input: topic_text: "{{topic_data.structured}}" genre: "{{topic_data.genre}}" - id: "gen_manga" # 並列B skill: "manga-pipeline" parallel_group: "content_gen" depends_on: ["analyze"] input: story_text: "{{topic_data.structured}}" genre_preset: "{{topic_data.genre}}" - id: "evaluate_all" # 両方完了後 skill: "content-evaluator" depends_on: ["gen_video", "gen_manga"] input: contents: ["{{gen_video.output}}", "{{gen_manga.output}}"] - id: "route" type: "condition_branch" depends_on: ["evaluate_all"] branches: - condition: "score >= 85" next: "publish" - condition: "score >= 70" next: "auto_improve" - condition: "score >= 55" next: "human_review" # HITL - condition: "default" next: "discard_retry"

Human-in-the-Loop

人間確認ポイントの
設計パターン

スコア55〜69のグレーゾーンや重要な判断を要する場合、 OpenClawがSignalで確認メッセージを送り、人間の返答を待ってから処理を続行します。 タイムアウトした場合のデフォルト動作も設定可能。

# HITL ステップ定義 (YAML) id: "human_review" type: "human_in_the_loop" channel: "signal" message_template: | ⚠️ 人間レビューが必要です コンテンツID: {{content_id}} スコア: {{score}} / 100 問題点: {{issues}} 👉 返信してください: [y] 公開する [n] 破棄する [e] 自動改善して再評価 timeout_minutes: 60 on_timeout: "discard" # 未返信は破棄 response_map: "y": "publish" "n": "discard_retry" "e": "auto_improve"

HITLが発動するシナリオ

グレーゾーン判定（スコア 55〜69）
AIが自信を持って判断できない場合。Signalで3択通知。
3回リトライ失敗
APIエラーが解消しない場合。詳細ログとともに通知。
コスト上限超過リスク
予想コストが制限の80%を超えた場合に事前確認。
安全性フラグ検知
著作権・規約違反の疑いがある場合は必ず人間確認。
APIコスト管理

全ツールのコストを
リアルタイムで一元管理

各APIの呼び出しコストをOpenClawが自動集計・記録します。 1ジョブあたり・1日あたりの上限を設定し、超過リスク時は自動で安価な代替ツールへフォールバック。 月次コストレポートをSignalで自動送信します。

  • 1ジョブのコストをリアルタイム積算
  • RunwayML高負荷時 → Kling AIへ自動切替
  • ElevenLabs超過時 → VOICEVOXへフォールバック
  • 日次・月次コストレポートをSignal送信
  • ツール別・ジャンル別のROI分析
# コスト管理設定 (openclaw_config.yaml) cost_management: budgets: per_job_usd: 2.00 per_day_usd: 30.00 per_month_usd: 600.00 fallbacks: "image.midjourney": "image.dalle3" "video.runwayml": "video.kling" "voice.elevenlabs": "voice.voicevox" "llm.gpt4o": "llm.claude-haiku" alerts: channel: "signal" thresholds: job_80pct: "warn_and_confirm" # HITL day_90pct: "pause_new_jobs" month_100pct: "full_stop" reporting: daily_summary: "08:00 JST" monthly_report: "1st 09:00 JST" include_roi: true # 収益 ÷ APIコスト

自律改善ループ

収益データがシステム全体を
自律的に進化させる

公開コンテンツの収益・再生数・エンゲージメントデータをOpenClawが自動収集し、 ネタ生成AI・評価閾値・生成パラメータへ反映します。 人間が何もしなくても、システムが週次で「何が売れるか」を学習し続けます。

1

収益データ収集

YouTube Analytics・note・Kindle APIから再生数・収益・CTRを毎日自動取得

2

パターン分析

上位20%の「ヒット作」と下位20%の「外れ作」の共通点をLLMで抽出

3

ネタAI更新

売れるジャンル・キーワード・トーンのスコアを更新。外れパターンは生成頻度を下げる

4

評価閾値調整

予測スコアと実収益のズレを補正。過剰な破棄・過剰な公開を自動チューニング

5

生成パラメータ最適化

スタイルプリセット・プロンプトテンプレートをヒット作に近づくよう自動更新
