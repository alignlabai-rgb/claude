W1
Phase 1: 基盤整備
Cowork設定 × Skillライブラリ × Files API — 「土台」を作る

Coworkアカウント設定（Day 1）

管理者がclaude.ai/teamでチームを作成。RBAC設定（MD=Admin, アナリスト=Member）。メンバー招待。

Group Memory初期投入（Day 2-3）

既存DDチェックリスト・バリュエーション基準・顧問リストをGroup Memoryに移行。チームで30分のKick-offで書き込みを依頼。

Skillライブラリ作成（Day 3-5）

.claude/agents/に財務・法務・ビジネスDDの3 Skillを作成。既存チェックリストを本文に変換するだけでよい。完璧を求めない。

Files API — DDデータルームのアップロード（Day 4-5）

進行中案件のPDFをFiles APIに一括登録。file_idをスプレッドシートで管理。以降のDDで再利用可能になる。

Week 1終了時点で得られるもの

チーム全員が同じSkillを使える環境が整う。新人が初日から「財務DDを開始して」と言えばチームの標準手順でDDが始まる。

Cowork RBAC Skills (.claude/agents/) Files API Group Memory
W2
Phase 2: マルチエージェント化
Agent Teams × MCP — 「並列処理」を手に入れる

settings.json にAgent Teams有効化（Day 1）

CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS: "1"を設定。OrchestratorとWorkerの役割分担を定義したdd-swarm-orchestrator.mdを作成。

社内DDデータベースMCPの構築（Day 2-4）

TypeScriptでdd-database MCPサーバーを実装。過去案件検索・バリュエーションベンチマーク取得の2ツールを先行リリース。Postgresに接続。

公式MCPサーバーの接続（Day 3）

Slack MCP + Google Drive MCPを設定ファイルに追加。DDレポートをDriveに自動保存 + Slackに要約通知するフローを構築。

DD Swarm初回試走（Day 4-5）

3エージェント並列DDを実際の案件で試走。Orchestratorの指示 → 3エージェントが並列実行 → 統合レポート生成 の全フローを確認。

Week 2終了時点で得られるもの

財務・法務・ビジネスDDが並列実行できる。従来3日かかったDD初期分析が午前中に完了するようになる。

Agent Teams Custom MCP (DD-DB) Slack MCP Google Drive MCP
W3
Phase 3: 自動化
Routines × Computer Use — 「呼ばなくても動く」状態へ

毎朝ニュースモニタリングRoutineの設定

毎朝7:50に起動してターゲット企業リストのニュースをスキャン。8:00に要約をSlack #ma-monitoring に投稿。/schedule "毎週月曜8時"で設定。

Computer Use — EDINET自動収集の構築

Dockerサンドボックスを用意。EDINETから対象企業の有価証券報告書を自動ダウンロード → Files APIにアップロード → 財務DDエージェントが即分析を開始するパイプラインを構築。

案件登録API Triggerの設定

社内の案件管理システムからWebhookでDD Swarmを起動。案件DBに新レコードが追加された瞬間にMCPがClaudeにPOSTしてDDが自動開始。

Advisor Tool の組み込み

日常処理はHaiku（エグゼキューター）、バリュエーション判断・重大リスク評価はOpus 4.6（Advisor）に委ねるハイブリッド構成に移行。コスト構造が変わる。

Week 3終了時点で得られるもの

毎朝出社するとSlackにターゲット企業の動向サマリーが届いている。案件登録の瞬間にDDが自動開始されている。「AIを動かす」から「AIが勝手に動いている」へ。

Phase 4: 最適化 & 拡張
Managed Agents × Prompt Caching × 継続改善 — 「本番品質」へ

Managed Agentsへの移行

Agent Teams実行環境をAnthropicクラウドに移行。サーバー管理ゼロ。自動Prompt Cachingが有効化されコスト最適化が完了。

Bloomberg MCP接続（優先度: 高）

Bloomberg Terminal APIをMCP経由でDDエージェントに接続。財務データ取得の人手作業が完全ゼロになる。

Skillライブラリの継続改善

毎案件終了後にSkillを更新。Group Memoryに学びを追記。3ヶ月でチームの集合知が蓄積され、新人が即日プロレベルのDDを実行できる状態になる。

SSEリモートMCPサーバーの本番化

社内サーバーにMCPをデプロイ。チーム全員がURLだけで同一ツールに接続。カスタムMCPが真の「チームインフラ」になる。
