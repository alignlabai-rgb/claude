1PicoClawとは？

PicoClawはSipeedが開発した超軽量AIアシスタントです。Go言語で書かれた単一バイナリで動作し、10MB未満のRAMと1秒以内の起動を実現。従来のOpenClaw（Python製）が約1GBのRAMを必要とするのに対し、99%のメモリ削減を達成しています。

コードの約95%はAIエージェント自身が生成する「自己ブートストラップ」手法で開発されました。1日で作られ、GitHubで1週間に12,000スターを獲得した話題のOSSです。

PicoClawでできること

🌐

Webリサーチ＆自動まとめ

毎朝ニュースを検索してTelegramに要約送信、定期レポート生成など。

🤖

ホームオートメーション

センサーデータを監視して異常をアラート。Raspberry Piで常時稼働。

💬

チャットボット常駐化

Telegram/Discord経由でどこからでもAIに話しかけられるパーソナルボット。

⏰

タスクスケジューラー

cronで定期実行。ファイル整理、GitHubイシューチェック、メール下書きなど。

📊

データ処理・変換

CSVの自動加工、APIからのデータ取得、レポート自動生成。

🛸

エッジAIデバイス

$10のLicheeRV-NanoやAndroid（Termux経由）でもAIエージェントが動作。

PicoClawはOpenClaw（旧称：OpenAI相当のOSS AIエージェント）の軽量代替です。

ハードウェアコスト$500以上推奨$10〜（LicheeRV-Nano）
言語PythonGo

3動作要件・対応機器

最小要件

  • RAM: 64MB（実使用 <10MB）
  • CPU: シングルコア 0.6GHz〜
  • OS: Linux（カーネル 4.9以上）
  • ストレージ: 20MB
  • インターネット接続

推奨環境

  • RAM: 256MB以上
  • CPU: 600MHz以上
  • OS: Debian / Ubuntu
  • ストレージ: 1GB以上
  • 安定したネット回線

対応プラットフォーム

🍓 Raspberry Pi Zero 2W 🍓 Raspberry Pi 3/4/5 ⚡ LicheeRV-Nano（$9.9・RISC-V） 📷 MaixCAM 🖥 NanoKVM 📱 Android（Termux経由） 💻 x86_64 Linux PC 🪟 Windows（WSL2） 🔷 ARM64 全般
💡 Windows ユーザー： WSL2（Windows Subsystem for Linux）を使えばWindowsでも動作します。Microsoft StoreからUbuntuをインストールしてください。

4インストール方法

✅ 方法①　バイナリ直接インストール（最速・5分）

アーキテクチャを確認してから対応するコマンドを実行してください。

x86_64（一般的なLinux PC・サーバー）

# バイナリをダウンロード
wget https://github.com/sipeed/picoclaw/releases/latest/download/picoclaw-linux-amd64
chmod +x picoclaw-linux-amd64
sudo mv picoclaw-linux-amd64 /usr/local/bin/picoclaw
picoclaw --version

ARM64（Raspberry Pi 3/4/5、その他ARMボード）

wget https://github.com/sipeed/picoclaw/releases/latest/download/picoclaw-linux-arm64
chmod +x picoclaw-linux-arm64
sudo mv picoclaw-linux-arm64 /usr/local/bin/picoclaw
picoclaw --version

RISC-V 64bit（LicheeRV-Nanoなど$10ボード）

wget https://github.com/sipeed/picoclaw/releases/latest/download/picoclaw-linux-riscv64
chmod +x picoclaw-linux-riscv64
sudo mv picoclaw-linux-riscv64 /usr/local/bin/picoclaw
picoclaw --version

自分のアーキテクチャを確認する方法

uname -m
# x86_64 → amd64 用を使用
# aarch64 → arm64 用を使用
# riscv64 → riscv64 用を使用

方法②　ソースからビルド（Go 1.21+ 必要）

git clone https://github.com/sipeed/picoclaw.git
cd picoclaw
make deps && make build
./picoclaw --version

方法③　Docker で起動

docker compose -f docker/docker-compose.yml --profile gateway up
# 設定ファイルを編集
vim docker/data/config.json
# バックグラウンドで再起動
docker compose -f docker/docker-compose.yml --profile gateway up -d
🐋 Dockerモードでは PICOCLAW_GATEWAY_HOST=0.0.0.0 を環境変数に設定しないとホストからアクセスできません。

5クイックスタート

初期化 
picoclaw onboard

対話形式で設定が進みます。APIキーの設定などを行います。

設定ファイルを確認・編集 
nano ~/.picoclaw/config.json
動作テスト 
picoclaw agent -m "こんにちは！何ができますか？"
# → AIが応答します
Webコンソールを起動（GUI） 
picoclaw gateway --port 18790
# ブラウザで http://localhost:18800 を開く

ブラウザベースのUIでチャット・設定が可能になります。

自動起動を設定（systemd） 
sudo nano /etc/systemd/system/picoclaw.service
[Unit]
Description=PicoClaw AI Agent
After=network.target

[Service]
ExecStart=/usr/local/bin/picoclaw agent
Restart=always
User=あなたのユーザー名
Environment=HOME=/home/あなたのユーザー名

[Install]
WantedBy=multi-user.target
sudo systemctl daemon-reload
sudo systemctl enable --now picoclaw
sudo systemctl status picoclaw

6設定詳細・ワークスペース構造

config.json の主要パラメータ

{
  "provider": "anthropic",
  "api_key": "sk-ant-xxxxxxxx",
  "model": "claude-sonnet-4-6",
  "max_tokens": 8192,
  "temperature": 0.7,
  "max_tool_iterations": 20,
  "request_timeout": 120,
  "restrict_to_workspace": true,
  "web_search": {
    "enabled": true,
    "provider": "duckduckgo"
  }
}

ワークスペースファイル（~/.picoclaw/workspace/）

PicoClawの「頭脳」を構成する特殊ファイルたちです。これらを編集することでAIの振る舞いをカスタマイズできます。

IDENTITY.md
エージェントの名前・役割・アイデンティティを定義
SOUL.md
性格・口調・価値観などパーソナリティ設定
USER.md
ユーザーの好みや背景情報（AIが学習・参照）
AGENTS.md
エージェントの行動ガイドライン・制約ルール
TOOLS.md
使用可能ツールの説明・使用条件
HEARTBEAT.md
30分ごとに実行される定期タスクのプロンプト
MEMORY.md
セッション間で持続する長期記憶
cron/
cronジョブの定義ファイル格納ディレクトリ
💡 HEARTBEAT.md の活用例： ここに「毎回起動時にTodo.txtを確認してタスクを提示せよ」と書いておくと、30分ごとに自動でタスクを確認してくれます。

10実践Tips集

  • まずDuckDuckGoから始める Web検索プロバイダーはDuckDuckGoが無料で使えます。Braveなどの有料APIが必要になるのは大量検索が必要な場合のみ。
  • restrict_to_workspace: true は必ず有効に デフォルトで有効ですが、外すとAIがシステム全体のファイルにアクセスできます。セキュリティのため有効のままにしましょう。
  • MEMORY.md でセッション間記憶を活用 「私の名前はXXです」などを MEMORY.md に書いておくと、毎回説明しなくてもAIが覚えてくれます。
  • HEARTBEAT.md で定期タスクを仕込む 30分ごとに実行されます。「毎回、今日のToDoリストを提示せよ」など常駐型エージェントとして活用できます。
  • temperature は用途で調整 コード生成なら 0.2〜0.4（正確さ重視）、創作や雑談なら 0.7〜0.9（多様性重視）が目安。
  • 起動しない時は --version で確認 picoclaw --version が動くならバイナリは正常。次に picoclaw agent -m "test" でAPIキーの問題かどうかを切り分けできます。
  • Webコンソールは外部公開しない ポート18800には認証がありません。外部からアクセスする場合はSSHトンネル（ssh -L 18800:localhost:18800 user@server）を使いましょう。
  • スキルはGitHubから直接インストール OpenClawのスキルエコシステムと互換性があります。picoclaw skill install github.com/xxx/yyy でインストール可能。
  • request_timeout は長めに設定 複雑なタスクでタイムアウトが起きる場合、デフォルトの120秒を300秒以上に増やすと安定します。
  • max_tool_iterations は控えめに デフォルトの20で十分なケースがほとんど。増やすとAPI費用も増えます。複雑なタスクだけ増やしましょう。

11よくある質問（FAQ）

PicoClawはWindowsで動きますか？
Windows用バイナリも配布されています（AMD64）。ただしLinuxが推奨環境です。WSL2上のUbuntuで動かすのが最も安定しています。
APIキーの費用はどれくらいかかりますか？
PicoClaw自体は無料ですが、LLMプロバイダーの費用がかかります。個人利用なら月$1〜5程度が目安です。DuckDuckGoでのWebサーチは無料です。
OpenClawのスキルはそのまま使えますか？
はい、GitHubからスキルを直接インストールする形でOpenClawのスキルエコシステムとの互換性があります。
起動しない・エラーが出る場合は？
①chmod +x picoclawで実行権限を確認、②file picoclawでアーキテクチャを確認、③picoclaw --versionでバイナリ動作確認、④config.jsonのAPIキーが正しいか確認、の順に試してください。
本番運用に使えますか？
現在v0.0.1の初期段階です。個人利用・実験・ホビープロジェクトには最適ですが、クリティカルな本番環境での使用は推奨されません。v1.0まで待つか、自己責任での利用が前提です。
「95%がAI生成コード」とはどういう意味？
PicoClawの開発過程で、コードの約95%はAIエージェント自身が生成しました（人間は監督・レビューのみ）。これが「自己ブートストラップ」と呼ばれる手法です。将来的にAIが自分のコードを改善するという新しい開発パラダイムの実証でもあります。
公式サポートはどこで受けられますか？
GitHub Issues（github.com/sipeed/picoclaw/issues）が主なサポート窓口です。Sipeedのコミュニティフォーラムやwikiも参照できます。
PicoClawトークン・コインがあると聞いたのですが？
それは詐欺です。PicoClawには公式のトークン・コインは一切存在しません。pump.funなどで見かけた場合は完全に無関係の詐欺プロジェクトです。公式ドメインはpicoclaw.ioのみです。

12用語集

OpenClaw
Anthropic等が開発したAIエージェントフレームワーク（Python製）。PicoClawはその超軽量代替版。
LLM
Large Language Model（大規模言語モデル）。Claude、GPT-4、Geminiなど。PicoClawはこれらを外部APIで利用する。
MCP
Model Context Protocol。AnthropicがAIとツールを連携させるために策定したオープンプロトコル。PicoClawはネイティブ対応。
RISC-V
オープンソースの命令セットアーキテクチャ（ISA）。LicheeRV-Nanoなど$10前後の超安価なボードで採用されている。
Heartbeat
PicoClawが30分ごとに自動実行する定期タスク。HEARTBEAT.mdにプロンプトを書いて設定する。
Workspace
PicoClawの作業領域（~/.picoclaw/workspace/）。セッション、メモリ、状態、cronジョブ、スキルを格納。
Soul / Identity
AIエージェントの性格・役割定義ファイル。SOUL.md・IDENTITY.mdを編集してキャラクターをカスタマイズできる。
Gateway
PicoClawのHTTPサーバー機能。チャットアプリ（Telegram等）との連携やWebコンソールに使用。デフォルトポートは18790/18800。
Skill
PicoClawに追加できる機能モジュール。GitHubから直接インストール可能。OpenClawのスキルと互換性あり。
Cron
時間を指定して定期的にコマンドやスクリプトを実行するUnixの仕組み。PicoClawもcron形式でタスクをスケジュールできる。
OpenRouter
複数のLLMプロバイダーを一つのAPIで利用できるサービス。Claude、GPT、Geminiなどを統一インターフェースで使える。
Termux
Android上でLinux環境を動かすアプリ。これを使うことでAndroidスマートフォンでPicoClawを動かせる。
