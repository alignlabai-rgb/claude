MCPの本質的な価値

MCPは「ツール定義をClaudeアプリから分離する」仕組み。社内DDデータベース接続MCPを一度作れば、Claude Code・Claude Desktop・Claude APIすべてで同じツールが使える。

実装例1: 社内DDデータベース検索MCP（TypeScript）

// dd-database-mcp/src/index.ts
import { McpServer } from "@modelcontextprotocol/sdk/server/mcp.js";
import { StdioServerTransport } from "@modelcontextprotocol/sdk/server/stdio.js";
import { z } from "zod";
import { Pool } from "pg";

const db = new Pool({ connectionString: process.env.DD_DATABASE_URL });

const server = new McpServer({
  name: "dd-database",
  version: "1.0.0",
});

// ツール1: 過去DD案件の検索
server.tool(
  "search_past_dd",
  {
    query: z.string().describe("検索キーワード（業種・規模・地域など）"),
    sector: z.string().optional().describe("業種フィルター"),
    year_from: z.number().optional().describe("開始年"),
  },
  async ({ query, sector, year_from }) => {
    const results = await db.query(
      `SELECT deal_name, sector, deal_size, ebitda_multiple,
              key_risks, lessons_learned, deal_date
       FROM past_deals
       WHERE to_tsvector('japanese', deal_name || ' ' || lessons_learned)
             @@ plainto_tsquery('japanese', $1)
       AND ($2::text IS NULL OR sector = $2)
       AND ($3::int IS NULL OR EXTRACT(YEAR FROM deal_date) >= $3)
       ORDER BY deal_date DESC LIMIT 10`,
      [query, sector || null, year_from || null]
    );

    return {
      content: [{
        type: "text",
        text: JSON.stringify(results.rows, null, 2)
      }]
    };
  }
);

// ツール2: バリュエーション前提の取得
server.tool(
  "get_valuation_benchmarks",
  {
    sector: z.string().describe("業種（例: 製造業, IT, 小売）"),
    deal_size: z.string().optional().describe("案件規模（small/mid/large）"),
  },
  async ({ sector, deal_size }) => {
    const benchmarks = await db.query(
      `SELECT metric, p25, median, p75, source, updated_at
       FROM valuation_benchmarks
       WHERE sector = $1
       AND ($2::text IS NULL OR deal_size_category = $2)
       ORDER BY metric`,
      [sector, deal_size || null]
    );

    return {
      content: [{
        type: "text",
        text: `${sector}のバリュエーション前提:\n` +
              JSON.stringify(benchmarks.rows, null, 2)
      }]
    };
  }
);

const transport = new StdioServerTransport();
await server.connect(transport);

# bloomberg-mcp/server.py
from mcp.server import Server
from mcp.server.stdio import stdio_server
from mcp.types import Tool, TextContent
import blpapi  # Bloomberg Python SDK

server = Server("bloomberg-data")

@server.list_tools()
async def list_tools() -> list[Tool]:
    return [
        Tool(
            name="get_financial_data",
            description="""Bloomberg APIから財務データを取得する。
            対象企業のティッカーシンボルと取得したい
            データフィールドを指定する。""",
            inputSchema={
                "type": "object",
                "properties": {
                    "ticker": {
                        "type": "string",
                        "description": "Bloomberg ticker (例: 7267 JP Equity)"
                    },
                    "fields": {
                        "type": "array",
                        "items": {"type": "string"},
                        "description": "取得フィールド (例: PX_LAST, EBITDA, TOT_DEBT)"
                    }
                },
                "required": ["ticker", "fields"]
            }
        )
    ]

@server.call_tool()
async def call_tool(name: str, arguments: dict):
    if name == "get_financial_data":

        fields = arguments["fields"]

本番運用
// SSEトランスポートでリモートMCPサーバーを構築
import { McpServer } from "@modelcontextprotocol/sdk/server/mcp.js";
import { SSEServerTransport } from "@modelcontextprotocol/sdk/server/sse.js";
import express from "express";

const app = express();
const server = new McpServer({ name: "dd-database-remote", version: "1.0.0" });

// ... ツール定義（stdioと同じ）...

// SSEエンドポイント
app.get("/sse", async (req, res) => {
  // Bearer tokenで認証
  const authHeader = req.headers.authorization;
  if (authHeader !== `Bearer ${process.env.MCP_SECRET_TOKEN}`) {
    return res.status(401).json({ error: "Unauthorized" });
  }

  const transport = new SSEServerTransport("/messages", res);
  await server.connect(transport);
});

app.listen(3000);
// デプロイ後: https://mcp.internal.yourbank.com/sse
// チームメンバーの設定ファイルに追加するだけで共有完了
{
  "mcpServers": {
    "dd-database": {
      "url": "https://mcp.internal.yourbank.com/sse",
      "headers": {
        "Authorization": "Bearer ${MCP_TOKEN}"
      }
    }
  }
}
セキュリティと統合
セキュリティ考慮事項 + Agent Teamsとの統合
MCPは強力だが、適切な認証・スコープ制限・監査ログが必須。

セキュリティチェックリスト

①

最小権限原則

MCPサーバーが持つAPI権限を最小限にする。DDデータ検索MCPに「DELETE権限」は不要。

②

認証トークンの環境変数化

APIキー・DBパスワードは設定ファイルにハードコードせずenvフィールドか環境変数で管理。

③

全ツール呼び出しのログ記録

MCPサーバー側でリクエスト・レスポンスのログを記録する。「ClaudeがDBに何を問い合わせたか」を監査可能に。

④

書き込み操作には確認ステップ

SlackへのメッセージPOST・DBへのINSERTなど副作用のある操作は人間の確認を挟む設計にする。

MCPとSkillsの使い分け

MCPは「外部ツールへの接続」、Skillsは「エージェントの役割・手順の定義」。MCPでBloombergから財務データを取得し、Skill（dd-financial-analyst）でそのデータを分析する。両者は補完関係にある。

echo '{"method":"tools/list","id":1}' | \
  node ./dd-database-mcp/dist/index.js

よくあるエラーと対処

Error: spawn npx ENOENT

Node.js/npmがインストールされていない、またはPATHが通っていない。which npxで確認。

ツールが一覧に出ない

設定ファイルのJSONが不正（末尾カンマなど）。json -c settings.jsonで構文確認。

API認証エラー

envのAPIキーが設定されていない。echo $GITHUB_TOKENで環境変数を確認。

Claudeがツールを使ってくれない

descriptionが不明確。「どういう時にこのツールを使うか」を日本語で明記する。
