Vibe Coding Handbook

Philosophy

コードを書く時代は終わりました。今は「知能を指揮する」時代です。

The Core Shift:
From "Writing Syntax" (文法を書く) → To "Directing Intelligence" (知能を指揮する)

Vibe Codingとは、AIエージェントをパートナーとして迎え、人間が「監督（Director）」兼「審美眼（Critic）」の役割に徹する開発スタイルです。あなたはもはやタイピストではありません。あなたはプロジェクトのオーナーであり、指揮官です。

The Mindset

❌ Old Way (The Typist)

  • 文法エラーを恐れる
  • すべての行を自分で理解しようとする
  • AIを「検索エンジンの代わり」として使う

✅ Vibe Way (The Director)

  • 「実現したい振る舞い」を語る
  • 詳細はAIに任せ、結果（Output）を評価する
  • AIを「専門家チームの部下」として扱う

The Vibe Loop

Vibe Codingの実践サイクルは、以下の4ステップを高速で回すことです。

1. Dictate (指揮)

「何を作りたいか」「なぜ必要か」「どんな雰囲気（Vibe）か」を言葉にします。
例: "ユーザー設定画面を作って。サイバーパンク風で、でも使いやすく。"

2. Generate (生成)

Agent (Claude Code / Copilot) がその意図をコードに変換します。彼らは数秒で数百行を実装します。

3. Verify (検証)

ブラウザやターミナルで結果を確認します。コードレビューは後回しです。「意図通り動いているか？」だけを見てください。

4. Refine (調整)

違和感をフィードバックします。「もっと暗くして」「アニメーションを速くして」。この繰り返しが品質を高めます。

The 7 Laws of Vibe

このスタイルを極めるための、絶対的な行動指針。

1. Don't Write, Dictate.

自分でコードを書こうとした瞬間、負けです。手は膝の上に。

2. Context is King.

「何を」より「なぜ」を伝えてください。背景情報こそが最大の武器です。

3. Trust but Verify.

AIを信頼しても、盲信してはいけません。必ず動かして確認すること。

4. Iterate Fast.

一発で正解を出そうとしない。60点のものを3秒で作らせ、それを磨くのです。

5. Read the Vibe.

コードの「行」ではなく、全体の「雰囲気（設計の匂い）」を感じ取ってください。

6. Be the CEO.

マイクロマネジメントはやめましょう。大きな方針を示し、細部は任せます。

7. Enjoy the Flow.

Vibe Codingは楽しいものです。苦しいと感じたら、やり方が間違っています。

Prompt Engineering

AIエージェントを動かすための「共通言語」。

❌ Bad Vibe (Instruction only)

ヘッダーの背景色を黒にして。あと文字を白にして。

✅ Good Vibe (Intent + Context)

サイト全体の印象を「高級感のあるモダンなダークモード」にしたい。
ヘッダーもそれに合わせて再設計して。配色は君のセンスに任せる。

Magic Phrases (魔法の言葉)

  • "Make it robust." (堅牢にして) → エラーハンドリングや型定義を強化してくれます。
  • "Refactor for readability." (読みやすくして) → 複雑なロジックを整理してくれます。
  • "Surprise me." (驚かせて) → AIの創造性を最大限に引き出し、予想外の提案をもらえます。
