コマ割りからキャラクター生成・セリフ・効果線まで、AIが一貫したスタイルで漫画を制作

最重要課題

キャラクターの
一貫性をどう保つか

AI漫画制作で最大の課題は「同じキャラクターを複数コマにわたって一貫して描くこと」。 毎回異なる顔になってしまうと漫画として成立しません。

  • LoRAモデルでキャラを事前学習
  • シード値・プロンプトの徹底管理
  • IP-Adapter でリファレンス画像を維持
  • キャラ設定シートをプロンプトに常時注入
  • 差分のみ生成して顔パーツを固定

制作フロー

6ステップで完成漫画へ

1

ストーリー構造化

ネタ文章をシーン・セリフ・感情・カメラアングルに構造化。LLMが漫画的な「間」や「演出」を考慮した構成に変換。

// 構造化出力例 { "scene": 3, "panels": [ { "angle": "close-up", "emotion": "surprise", "dialogue": "えっ、本当に？", "sfx": "ドキッ" } ] }
2

コマ割り設計

ページあたりのコマ数・サイズ・配置をAIが自動設計。テンション高い場面は大ゴマ、説明場面は小ゴマを自動判断。見開きや見せ場コマも自動挿入。

GPT-4o (構成判断) カスタムレイアウトエンジン
3

キャラクター・背景生成

LoRA + IP-Adapterで一貫したキャラを生成。背景はControlNet + 深度マップで遠近感を保持。白黒漫画風・カラー漫画風の切り替えも対応。

Stable Diffusion XL ControlNet IP-Adapter ComfyUI
4

吹き出し・セリフ配置

セリフの長さ・感情に合わせた吹き出し形状を自動選択（通常・叫び・思考・ナレーション）。フォント・サイズ・配置をコマ内の空白スペースに最適化。

Pillow (Python) カスタム吹き出しSVG 自動レイアウト調整
5

効果線・トーン追加

スピード線・集中線・背景トーン（スクリーントーン）を自動追加。シーンの感情に合わせた効果を自動判断して適用。

カスタムComfyUIノード OpenCV (線処理)
6

ページ結合・出力

全コマをページにレイアウトし、PDF・CBZ・Web閲覧用HTMLとして出力。電子書籍配信用のメタデータも自動生成。評価システムへ送付。

PDF出力 Kindle MOBI/EPUB Web漫画HTML

ComfyUIワークフロー構成

ComfyUIをバックエンドとして使用し、各生成ノードをAPIで制御。キャラクター別にLoRAを管理し、シーンごとに自動切替。

  • ComfyUI APIでバッチ生成を自動化
  • キャラ×感情×角度のプロンプトマトリクス管理
  • ControlNet Cannyで線画の一貫性を確保
  • ADetailerで顔の品質を自動修正
  • 生成失敗時の自動リトライ機能
