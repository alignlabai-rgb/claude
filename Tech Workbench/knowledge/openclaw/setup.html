🏰 Pattern A: The Fortress (WSL 2)

Windowsユーザーにとっての黄金比。母艦を汚さず、Linuxのパワーをフル活用できます。
ファイルシステムアクセスが高速で、VS Codeとの統合も完璧です。

PerformanceNative Speed
IsolationHigh (VM)
ToolingExcellent (VS Code)

Step 1: Install WSL

PowerShellを管理者権限で開き、以下のコマンドを実行します。

# インストール（Ubuntuがデフォルトで入ります） $ wsl --install # ここでPCを再起動してください # 再起動後、自動的にUbuntuのセットアップ画面が開きます

Step 2: Setup Environment (in Ubuntu)

Ubuntuのターミナルを開き、以下の手順でNode.js環境を作ります。

# システムの更新 $ sudo apt update && sudo apt upgrade -y # Node.js (Latest LTS) のインストール $ curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash - $ sudo apt install -y nodejs build-essential # インストール確認 $ node -v > v20.x.x

Step 3: Install OpenClaw

$ npm install -g @openai/openclaw # 動作確認 $ openclaw --version
📦 Pattern B: The Clean Room (DevContainer)

VS Codeの「Remote - Containers」機能を使い、プロジェクトごとに環境を完全に隔離します。
「このプロジェクト専用のOpenClawバージョン」を固定したい場合に最適です。

Step 1: Prerequisites

  • Docker Desktop がインストールされ、起動していること。
  • VS Code に "Dev Containers" 拡張機能が入っていること。

Step 2: Create Configuration

プロジェクトのルートディレクトリに設定ファイルを作ります。

# フォルダ作成 $ mkdir .devcontainer $ touch .devcontainer/devcontainer.json

Step 3: Edit devcontainer.json

以下の内容をコピペしてください。

{ "name": "OpenClaw Env", "image": "mcr.microsoft.com/devcontainers/typescript-node:20", "features": { "ghcr.io/devcontainers/features/git:1": {} }, "customizations": { "vscode": { "extensions": [ "dbaeumer.vscode-eslint", "esbenp.prettier-vscode" ] } }, "postCreateCommand": "npm install -g @openai/openclaw && echo 'OpenClaw Ready!'" }

Step 4: Launch

VS Code 左下の緑のボタン（><）をクリックし、"Reopen in Container" を選択します。
初回はコンテナのビルドに数分かかりますが、次回からは一瞬で起動します。

🪟 Pattern C: Windows Native

WSLやDockerを使わず、直接PowerShellで動かす方法。
手軽ですが、Linux専用コマンド（chmod, grepなど）が使えないため、Mission作成時に注意が必要です。

Step 1: Install Node.js

Node.js 公式サイト からインストーラー（LTS版）をダウンロードし、インストールします。

Step 2: Install via PowerShell

# バージョン確認 PS C:\> node -v # OpenClawインストール PS C:\> npm install -g @openai/openclaw

Step 3: Execution Policy

Windowsセキュリティにより、デフォルトではスクリプト実行がブロックされます。

# 現在のエラーを確認 PS C:\> openclaw # -> "スクリプトの実行が無効になっているため..." # 権限を変更（現在のユーザーのみ許可） PS C:\> Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
⚠️ 注意点: エージェントには「PowerShellコマンドを使ってください」と明示的に指示する必要があります。
例: "Constraints: Use PowerShell syntax (e.g. `ls`, `cat` are aliases, but `grep` is `sls`)."
🍎 Pattern D: Docker for Mac

メイン機のMacを「汚染」から守るためのアプローチ。
エージェント用のサンドボックスを一瞬で作り、終わったら破棄します。

Step 1: Install Docker Desktop

Docker公式サイトから "Docker Desktop for Mac (Apple Silicon / Intel)" をインストールします。

Step 2: Create Alias

毎回長いコマンドを打つのは大変なので、エイリアスを作ります。

# ~/.zshrc に追記 alias openclaw="docker run -it --rm \ -v \$(pwd):/app \ -w /app \ -e OPENAI_API_KEY=\$OPENAI_API_KEY \ node:20-slim \ npx @openai/openclaw" # 反映 source ~/.zshrc

Step 3: Run

# どのディレクトリでも実行可能 $ cd my-project $ openclaw run mission.md

このコマンドを実行すると、一時的なコンテナが立ち上がり、現在のフォルダ（`my-project`）だけがマウントされます。
もしエージェントが `rm -rf /` を実行しても、消えるのはコンテナの中身だけで、あなたのMacは安全です。

☁️ Pattern E: The Factory (Cloud VPS)

月額500円程度のVPS (Vultr, DigitalOcean, ConXH) で24時間稼働させる、プロの運用。
「寝ている間に」すらPCを立ち上げておく必要がなくなります。

Step 1: Setup VPS

Ubuntu 22.04 LTS の最小構成インスタンスを作成し、SSH接続します。

Step 2: Install Environment

$ ssh root@YOUR_IP # 必要なツールを一括インストール $ apt update && apt install -y nodejs npm git $ npm install -g n $ n lts

Step 3: Process Management (PM2)

SSHを切ってもエージェントが死なないように、PM2（プロセスマネージャー）を使います。

$ npm install -g pm2 @openai/openclaw # バックグラウンド実行の例 $ pm2 start "openclaw run mission.md" --name daily-task # ログ監視 $ pm2 logs daily-task

Expert Tip: Cron Job

「毎朝4時に実行」などのスケジュール実行も可能です。

$ crontab -e # 毎日AM 4:00に実行 0 4 * * * /usr/bin/node /usr/lib/node_modules/.bin/openclaw run /root/tasks/daily-report.md
