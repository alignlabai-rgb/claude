MCP & Tools

Extending Availability: Creating Custom Tools

既存のMCPサーバーを使うだけでは不十分です。
真のMasteryは、「自分の業務専用のツール」を自作し、Claudeに与えることで達成されます。

Build Your Own MCP Server

例えば、「社内の用語集APIを検索するツール」をClaudeに持たせる手順です。

01

Initialize Project

MCPサーバーは単なるNode.js (またはPython) アプリです。
TypeScriptのSDKを使うのが最も簡単です。

$ mkdir my-mcp-server && cd my-mcp-server $ npm init -y $ npm install @modelcontextprotocol/sdk zod
02

Implement the Tool

index.ts を作成し、「ツール定義」と「実行ロジック」を書きます。

import { McpServer } from "@modelcontextprotocol/sdk/server/mcp.js"; import { z } from "zod"; const server = new McpServer({ name: "InternalGlossary", version: "1.0.0" }); // ツールを登録 server.tool( "search-glossary", { query: z.string().describe("検索したい社内用語") }, async ({ query }) => { // ここで実際のAPIやDBを叩く const result = `用語: ${query} の定義は...`; return { content: [{ type: "text", text: result }] }; } ); const transport = new StdioServerTransport(); await server.connect(transport);
03

Connect to Claude

Claude Code にこのツールを認識させます。claude_config.json に追記するだけです。

{ "mcpServers": { "my-glossary": { "command": "node", "args": ["/path/to/my-mcp-server/dist/index.js"] } } }

これで、Claudeに対して「社内用語の『プロジェクトX』って何？」と聞くだけで、自動的にこのツールが呼ばれ、正しい答えが返ってくるようになります。

Best Practices

  • Atomic Tools: 1つのツールには1つの機能だけを持たせる。「検索して更新する」ではなく「検索」と「更新」に分ける。
  • Rich Description: describe("...") の記述が全てです。ここに「どういう時に使うべきか」を詳しく書くと、Claudeが適切にツールを選んでくれます。
