システム内での役割

品質ゲートとして
全コンテンツを審査

評価システムは動画・漫画の両パイプラインから成果物を受け取り、 公開可否を判定するゲートキーパーです。 評価結果はマネタイズシステムへ転送され、低品質コンテンツをフィルタリングします。 さらに評価データをネタ生成AIへフィードバックし、システム全体を自律改善します。

  • 入力元: 動画パイプライン / 漫画パイプライン
  • 出力先: マネタイズシステム（合格時）/ 再生成キュー（不合格時）
  • フィードバック先: ネタ生成AI・LLMプロンプト最適化
  • 判定速度: 1コンテンツあたり 約3〜5秒
  • 自動改善: スコア70〜84は改善指示とともに再処理

モジュール入出力定義

# OpenClaw スキル定義 (SKILL.md 抜粋) name: content-evaluator version: "3.0.0" input: content_type: string # video|manga file_path: string # 評価対象ファイル metadata: object # タイトル/説明/タグ target_platforms: list output: total_score: float # 0〜100 decision: string # PUBLISH|IMPROVE|REVIEW|DISCARD scores: object # 指標別スコア improvements: list # 改善指示リスト feedback_data: object # ネタAIへのフィードバック

スコアリング指標

10項目の多角的評価

📊
重み: 20%

エンゲージメント予測スコア

過去のバイラルコンテンツデータを元に、視聴継続率・いいね率・コメント率を予測。類似コンテンツとのパフォーマンス比較も実施。

🎨
重み: 15%

ビジュアル品質スコア

画像の解像度・構図・色調の一貫性・アーティファクトの有無をVision AIで自動チェック。漫画の場合はコマ割りの読みやすさも評価。

✍️
重み: 15%

コンテンツ品質スコア

ストーリーの構成・セリフの自然さ・情報の正確性・オリジナリティをLLMが評価。剽窃チェックも同時実行。

🔍
重み: 15%

SEO・発見可能性スコア

タイトル・説明文・タグのSEO最適度を評価。Googleトレンド・YouTube検索ボリュームとの照合でタイムリー性も判定。

🛡
重み: 10%

安全性・規約遵守スコア

著作権侵害リスク・プラットフォームガイドライン違反・有害コンテンツを事前検知。問題がある場合は自動で除外フラグ。

🎯
重み: 10%

ターゲット整合性スコア

想定ターゲット層への訴求度を評価。年齢層・趣味・ライフスタイルとコンテンツの整合性を確認。

📈
重み: 5%

トレンドスコア

現在のSNSトレンド・話題のキーワードとの関連性を評価。時事ネタの有効期限も考慮したタイムリー性判定。

🔄
重み: 5%

シリーズ継続性スコア

既存シリーズとの一貫性・次回への引き・視聴者の離脱ポイントを予測。シリーズ化の可能性も評価。

判定フロー

スコアに基づく自動判定

85+

即時公開

品質確認済み。マネタイズパイプラインへ自動送付し、最適な時間帯に投稿予約。

70-84

改善して公開

低スコア項目を特定し、AI自動改善を実施。再評価後に公開判定。

人間レビュー

判断が難しいグレーゾーン。最終判断を人間に委ねてレビューキューに追加。

<55

破棄・再生成

品質基準未達。ネタを再投入して別のアプローチで再生成。失敗パターンをDBに記録。

連携設計

評価結果の
マルチパス転送設計

評価エンジンはスコアに応じて3つの異なるパスでコンテンツを処理します。 合格コンテンツはマネタイズシステムへ、 改善可能なコンテンツはAI自動修正ループへ、 失敗パターンはネタ生成AIの学習データへフィードバックされます。

# 評価後ルーティング (OpenClaw YAML) on_evaluation_complete: - condition: "score >= 85" action: forward_to_monetization skill: publish-scheduler - condition: "score >= 70" action: auto_improve skill: content-improver then: re_evaluate - condition: "score >= 55" action: human_review_queue notify: signal_message - condition: "score < 55" action: discard_and_feedback skill: failure-analyzer # ネタAIへ

評価エンジンの構成

# 評価エンジン (Python) class ContentEvaluator: def evaluate(self, content): scores = {} # 並列で各スコアを計算 with ThreadPoolExecutor() as ex: futures = { ex.submit(self.visual_score, content): "visual", ex.submit(self.content_score, content): "content", ex.submit(self.seo_score, content): "seo", ex.submit(self.safety_score, content): "safety", ex.submit(self.engagement_predict, content): "engagement", } for future, name in futures.items(): scores[name] = future.result() # 重み付き合計スコア算出 total = self.weighted_score(scores) return EvaluationResult(scores, total)
