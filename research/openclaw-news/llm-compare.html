2. 各LLMの詳細分析

🍊
Claude Sonnet 4.6（Anthropic）
OpenClaw開発者も推奨。指示遵守と日本語品質が群を抜く
✅ 強み
  • SKILL.mdの指示をほぼ完全に遵守。意図しない動作が最も少ない
  • 200Kトークンのコンテキスト長。長大なメール履歴や文書も一括処理
  • 日本語の自然さがトップクラス。敬語・ビジネス文書の作成も得意
  • 安全性への取り組みが業界最高水準（Constitutional AI）
⚠️ 弱み
  • コストが中程度（DeepSeekの約11倍）
  • APIのレートリミットが厳しく、高頻度タスクで詰まることがある
  • 時々「安全上の理由」で正当なタスクを断ることがある
📝 文書作成・要約
📧 メールの高度な分類・返信
🧠 複雑な意思決定支援
📊 長文レポート分析
💰 コスト重視タスクには不向き
👤 こんな人に最適： 日本語での高品質な文書処理・複雑なビジネスタスクを自動化したい人。コストより品質を重視する場合。
🔵
DeepSeek V3（DeepSeek AI）
コスト効率ナンバーワン。試験運用・大量処理に最適。プライバシーに注意
✅ 強み
  • Claude比で約1/11のコスト。試行錯誤・大量処理に圧倒的有利
  • 推論能力はGPT-4oに匹敵するレベルに到達（2026年初頭）
  • コーディングタスクに特に強い（HumanEval等のベンチマークでトップクラス）
  • 応答速度が速い
⚠️ 弱み・注意点
  • 中国企業のサービス。 機密情報・個人情報は送らないこと
  • 高負荷時（特に中国の昼間）にAPIが不安定になることがある
  • SKILL.mdの複雑な指示の遵守精度がClaudeより若干低い
  • 政治・社会的に敏感なトピックに制限あり
🔴 プライバシー注意： DeepSeekは中国企業（深セン市）が運営。APIを通じて送信したデータは中国のサーバーを経由する可能性がある。業務データ・個人情報の処理には使用しないこと。
🧪 プロトタイプ・試験運用
💻 コード生成・レビュー
📰 公開情報の大量処理
🔒 機密情報の処理はNG
🏢 業務データの処理はNG
👤 こんな人に最適： まずOpenClawを試してみたい・低コストで大量のスキルを実験したい人。公開情報の処理に限定する場合。
🟢
GPT-4o（OpenAI）
API安定性トップ。マルチモーダル対応。エンタープライズに信頼性高い
✅ 強み
  • API安定性・稼働率が業界最高水準。24時間365日の自律エージェントに最適
  • 画像・音声・テキストのマルチモーダル対応。スクリーンショット分析も可能
  • OpenAIのエコシステムとの統合が豊富（DALL-E、Whisper等）
  • Function Calling（ツール呼び出し）の精度が高く、Skillとの相性が良い
⚠️ 弱み
  • コストがやや高め（DeepSeekの約9倍）
  • コンテキスト長がClaudeの半分（128K）
  • Claudeと比べると指示の微細なニュアンス遵守がやや劣る
🖼️ 画像・スクショ分析
🏢 エンタープライズ用途
🔄 24時間連続稼働
🔗 OpenAIエコシステム活用
👤 こんな人に最適： 安定性を最優先にしたい・マルチモーダル機能を活用したい・すでにOpenAIエコシステムを使っている人。
🟡
GPT-4o mini（OpenAI）
日常の定型タスクに最適なコスパ最強モデル
✅ 強み
  • 主要モデル中最安値クラス（$0.15/1Mトークン）
  • 応答速度が最速クラス
  • API安定性はGPT-4oと同等
  • 定型・反復タスクには十分な性能
⚠️ 弱み
  • 複雑な推論・多段階タスクで精度が落ちる
  • SKILL.mdの細かい指示を取りこぼすことがある
  • 長い文章の要約精度がやや劣る
📅 カレンダー管理
📬 簡単なメール分類
⏰ リマインダー設定
🔔 アラート通知
🧠 複雑な推論は不向き
👤 こんな人に最適： 毎日動く定型自動化を低コストで実現したい人。メール分類・リマインダー・シンプルなデータ収集など。

3. おすすめの使い分け戦略

🎯 タスク別LLM選択ガイド
タスクの種類 推奨LLM 理由
📝 高品質な文書作成・要約 Claude Sonnet 日本語品質・指示遵守が最高
🧪 新しいSkillの試験運用 DeepSeek V3 コスト1/11。失敗してもダメージ小
🖼️ 画像・スクリーンショット分析 GPT-4o マルチモーダル対応が必須
📅 毎日の定型自動化 GPT-4o mini 最安値×高安定性の組み合わせ
🔒 機密情報を含むタスク Claude Sonnet Anthropicのプライバシーポリシーが最も厳格
💻 コード生成・デバッグ DeepSeek V3 コーディングベンチマークでトップクラス×低コスト

💡 プロのテクニック：タスクでLLMを使い分ける設定

# ~/.openclaw/config.yaml でSkillごとにLLMを変える skills: email-sorter: llm: gpt-4o-mini # 定型タスクは安価なモデルで document-analysis: llm: claude-sonnet-4-6 # 複雑な分析は高性能モデルで code-review: llm: deepseek-v3 # コードは低コストで十分 image-analyzer: llm: gpt-4o # 画像処理はGPT-4oのみ対応
