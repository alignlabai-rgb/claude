👥 Claude Cowork · 2026年4月 GA

Claude Cowork

「自分だけのClaude」から「チーム全員のClaude」へ。RBAC・Group Memory・共有Projects・コスト管理ダッシュボードで、AIをチームの公式インフラに変える。2026年4月、一般公開（GA）。

ステータス: GA（2026年4月）
対応プラン: Team / Enterprise
管理画面: claude.ai/team
対象: 5名〜組織全体
基本概念
Coworkとは何か — チームAIインフラへの転換
個人がバラバラにClaudeを使う状態から、チームで統制・共有・最適化する状態へ。
❌ Before: 個人がバラバラに使う
  • 各自がPersonalプランで各自払い
  • 誰が何にClaudeを使っているか管理者が把握できない
  • 過去のDDナレッジは個人のチャット履歴に埋もれる
  • 新人が入っても引き継げる資産がない
  • コスト管理・セキュリティポリシーの適用が不可能
  • 同じ質問を全員が個別に繰り返す
✅ After: Coworkでチーム管理
  • 一括契約でコスト最適化・領収書一本化
  • 管理ダッシュボードで全員の利用状況を可視化
  • Group Memoryにチームの共有ナレッジを蓄積
  • 新人が初日からチームのナレッジにアクセス可能
  • RBACで機密情報へのアクセスを役割別に制御
  • 共有Projectsで案件ごとにチームが協力
主要機能
Coworkの4つのコア機能
2026年4月GA版で利用可能な機能一覧。
🔐 機能1 — RBAC（役割ベースアクセス制御）

誰が何にアクセスできるかを役割で管理する

金融機関において最も重要な機能。案件情報・DDレポート・ターゲットリストへのアクセスを 役割（Role）に基づいて制御する。「アナリストはPublicプロジェクトのみ」「MDは全案件参照可」 といった設定が管理画面から数クリックで完了する。

👑 Admin（管理者）

MD / 部長クラス

  • 全プロジェクト・全メモリへのアクセス
  • メンバー招待・権限変更
  • コスト上限・ポリシー設定
  • 利用ログの全閲覧
👤 Member（メンバー）

アナリスト / アソシエイト

  • 担当プロジェクトへのアクセス
  • Group Memoryの読み取り・書き込み
  • 共有Skillsの利用
  • 自分の利用ログのみ閲覧
👁️ Viewer（閲覧者）

外部顧問 / インターン

  • 指定プロジェクトの閲覧のみ
  • Group Memoryは読み取り専用
  • Skillsの利用（書き込み不可）
  • ログエクスポート不可
🧠 機能2 — Group Memory（チーム共有記憶）

チームの知識を蓄積し、全員が恩恵を受ける

個人のPersonal Memoryとは別に、チーム全員が共有する記憶領域。 「先週のDDで学んだXX業界の注意点」「過去に失敗した交渉パターン」 「よく使うバリュエーション前提」を蓄積すると、次の案件から全員が活用できる。

MAチームが蓄積すべきGroup Memory

業界ナレッジ

セクター別DDチェックポイント

「製造業のDDではWCAPの季節変動に注意。特にXX業種は在庫評価に独自リスクがある」

バリュエーション前提

業界別EV/EBITDAレンジ（更新日付き）

「2026年Q1時点の国内製造業M&AのEV/EBITDA中央値は8.2x（出典: XX）」

過去案件パターン

見落としがちなリスクパターン集

「環境負債はオフバランスになりやすい。製造業・化学系では必ず土壌汚染リスクを確認」

アドバイザー情報

法務・税務顧問の得意分野メモ

「XX法律事務所のYY弁護士はCross-border案件に強い。ターンアラウンド1週間以内」

Group Memoryへの書き込み方法

# Claude Codeから直接書き込み
"このDDで学んだことをチームのGroup Memoryに追加してください:
XX製造業は売掛金の回収期間が業界平均より長い傾向がある。
DDでは売掛金明細（顧客別・期日別）を必ず入手すること。"

# 管理画面（claude.ai/team/memory）からも追加可能

# Routinesで定期的に更新
"先週完了したDDの知見をGroup Memoryに追記してください"
ナレッジマネジメントのコツ

案件終了のたびに「次のDDで活かせる学び」を3つGroup Memoryに書き込む習慣をつける。半年でチームの集合知が形成される。

📁 機能3 — 共有Projects（チーム協働作業スペース）

案件ごとにチームが同一コンテキストで協力する

個人のProjectsとは別に、チーム全員がアクセス・編集できる共有Projectsを作成できる。 DDプロジェクトごとに専用Projectを立ち上げ、システムプロンプト（案件概要・指示）・ アップロードファイル・チャット履歴をチームで共有する。

  • Projectシステムプロンプト: 案件の基本情報・対象企業概要・DDの目的を設定→全員が同じ前提で作業
  • ファイル共有: DDソース資料をProjectにアップロード→チーム全員がFiles APIのように参照可能
  • チャット履歴の共有: 他のメンバーのClaude対話を見て学習・引き継ぎがスムーズに
  • Skillsとの統合: Project専用のSkillセットを設定（このProjectではdd-financial-analystを使用）
📊 機能4 — 利用ダッシュボード（コスト・ログ管理）

誰が何に使っているかを可視化・管理する

金融機関のコンプライアンス要件を満たすための管理機能。 誰がいつClaudeに何を入力したかのログ、部門・プロジェクト別のコスト配賦、 上限設定による予算管理まで一元管理できる。

  • メンバー別・プロジェクト別のトークン使用量とコスト
  • 月次・週次のコストレポート（経費申請・部門配賦に使える）
  • コスト上限アラート（月$XXを超えたら管理者に通知）
  • 監査ログ: 全入出力の記録（エクスポート可）
  • コンプライアンスモード: 機密キーワードを含む入力の検出・アラート

RBAC設計案（MAチーム）

MD

Admin: マネージングディレクター

全案件・全メモリ・コスト管理。Skillライブラリの更新権限。監査ログエクスポート。

担当

Member: 担当アナリスト / アソシエイト

担当案件のProjectにフルアクセス。Group Memoryへの書き込み可。Skillsの利用可。

外部

Viewer: 外部顧問 / インターン

指定Projectの閲覧のみ。Group Memoryは読み取り専用。機密情報への直接アクセスなし。

導入ロードマップ

W1

管理者セットアップ（claude.ai/team）

チームアカウント作成・RBAC設定・メンバー招待メール送付。所要時間: 約2時間。

W1

Group Memory初期コンテンツの投入

既存のDDチェックリスト・ガイドライン・ノウハウをGroup Memoryに移行。メンバーに書き込みを依頼。

W2

Skillライブラリの共有設定

既存の.claude/agents/をチームリポジトリに移動。共有Projectsと連携設定。

W2-3

進行中案件をCowork Projectsに移行

案件ごとに共有Projectを作成。システムプロンプト（案件概要）・DDソース資料をアップロード。

M2〜

Routines・Agent Teamsとの統合

Coworkの共有ProjectsをDD Swarmの起点に設定。毎朝の自動モニタリングをチーム全員が確認できる状態に。

統合方法
既存ワークフローへの統合
CoworkはAPIとSlack連携で既存ツールとシームレスに接続できる。

Slack連携（通知・起動）

# Cowork API + Slack Bot連携例
import anthropic
from slack_sdk import WebClient

claude = anthropic.Anthropic()
slack = WebClient(token=SLACK_BOT_TOKEN)

def handle_slack_command(command: str, user: str, channel: str):
    """SlackコマンドでCoworkプロジェクトのDDを起動"""

    # Coworkのチームプロジェクトを指定してClaude実行
    response = claude.messages.create(
        model="claude-sonnet-4-6",
        max_tokens=4096,
        system=get_project_system_prompt("ma-dd-2026"),
        # Group Memoryは自動的に参照される（Team版）
        messages=[{"role": "user", "content": command}]
    )

    # 結果をSlackに投稿
    slack.chat_postMessage(
        channel=channel,
        text=f"*DD分析結果* (@{user})\n{response.content[0].text}"
    )

# Slackで "/dd XYZ企業の財務サマリーを出して" と入力するだけで実行
Enterprise版の追加機能

Enterprise版ではSAML/SSOによる社内認証基盤との統合、コンプライアンスフィルタリング（機密キーワードの検出・ブロック）、専用クラウドリージョン（日本リージョン対応）が追加される。金融機関向けの要件に対応。

次のステップ

Coworkは「AIの民主化」と「ガバナンス」を両立する仕組み。まず管理者（MD）がアカウントを作成し、小さく始めること。5人×1ヶ月の試用で、Group Memoryにどれだけのナレッジが蓄積されるかを測ると導入価値が実感できる。
