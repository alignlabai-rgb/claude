📦 インストール・起動

Q インストールしたが openclaw: command not found と出る
🔴 よくあるエラー

PATHが通っていない可能性があります。以下を順番に試してください。

# 1. インストール先を確認 pip show openclaw | grep Location # 2. PATHにPythonのbinディレクトリを追加 # ~/.bashrc または ~/.zshrc に追記: export PATH="$HOME/.local/bin:$PATH" # 3. 設定を反映 source ~/.bashrc # または source ~/.zshrc # 4. 再確認 which openclaw
💡 Mac + zsh の場合： ~/.zshrc を編集してください。~/.bashrc では反映されないことがあります。
Q Dockerで起動するとすぐ終了する

APIキーが正しく設定されていないことが多い原因です。ログを確認します。

# ログを確認（終了したコンテナのログも見られる） docker logs openclaw # よくあるエラーメッセージ例: Error: ANTHROPIC_API_KEY is not set Error: config.yaml not found at /root/.openclaw/config.yaml # 対処: 環境変数を正しく渡す docker run -d \ -v ~/.openclaw:/root/.openclaw \ -e ANTHROPIC_API_KEY="sk-ant-api..." \ --name openclaw \ ghcr.io/openclaw/openclaw:latest
Q Windowsでインストールしようとするとエラーになる

WindowsではWSL2（Windows Subsystem for Linux）を使うのが最も安定します。

# 1. PowerShellを管理者で実行してWSL2をインストール wsl --install # 2. 再起動後、Ubuntu を開いて通常通りインストール sudo apt update && sudo apt install python3-pip pip install openclaw
⚠️ PowerShellでのネイティブ動作は非推奨。WSL2環境で動かすことを強く推奨します。
Q ModuleNotFoundError が大量に出る

依存関係のインストールが不完全です。仮想環境を使うと解決することが多いです。

# venvを使う方法 python3 -m venv ~/.venv/openclaw source ~/.venv/openclaw/bin/activate pip install -r requirements.txt openclaw start # または uvx（高速・依存関係自動解決） pip install uvx uvx openclaw
Q 起動はするが Telegram/Discord ボットが反応しない

ボットトークンの設定ミスか、ボットへのアクセス権限の問題です。

# よくある原因チェックリスト: # □ config.yaml の messaging.token が正しいか確認 # □ Telegramの場合: ボットに /start を送ったか # □ Discordの場合: ボットをサーバーに招待したか # 接続テスト openclaw test --messaging # ログでエラーを確認 openclaw logs --filter messaging
💡 Telegramの場合： @BotFather で発行したトークンの先頭に余分なスペースが入っていないか確認してください。コピペミスが多い原因です。

🔑 APIキー・LLM接続

Q AuthenticationError: Invalid API Key が出る
🔴 よくあるエラー
# APIキーの確認方法（Anthropicの場合） curl -s https://api.anthropic.com/v1/messages \ -H "x-api-key: YOUR_KEY" \ -H "anthropic-version: 2023-06-01" \ -H "content-type: application/json" \ -d '{"model":"claude-haiku-4-5-20251001","max_tokens":10,"messages":[{"role":"user","content":"hi"}]}' # 正常なら応答が返る。エラーなら以下を確認: # 1. キーが "sk-ant-api03-..." で始まるか # 2. キーの前後に余分なスペース・改行がないか # 3. Anthropicコンソールでキーが有効か確認 # 4. アカウントに支払い情報が登録されているか
Q 途中でレスポンスが止まる・タイムアウトする

APIのレートリミットに達しているか、ネットワーク問題の可能性があります。

# config.yaml でタイムアウトとリトライを設定 llm: timeout: 60 # タイムアウト秒数（デフォルト30） max_retries: 3 # リトライ回数 retry_delay: 5 # リトライ間隔（秒） # レートリミットエラーの場合は「低速モード」を試す requests_per_minute: 10 # デフォルトは制限なし
Q DeepSeekを設定したが接続できない
# DeepSeek の設定例（config.yaml） llm: provider: deepseek model: deepseek-chat # DeepSeek V3 api_key: sk-xxxxxxxxxxxxxxxx base_url: https://api.deepseek.com # 必須 # DeepSeek R1（推論モデル）の場合 model: deepseek-reasoner
⚠️ DeepSeekは中国からのアクセスが多い時間帯（日本時間の午前中）に応答が遅くなることがあります。繰り返しタイムアウトする場合はその時間帯を避けるか、ClaudeやOpenAIに切り替えてください。
Q SkillごとにLLMを変えるにはどうすればいい？

config.yaml で Skill 別にモデルを指定できます。

# config.yaml — デフォルトと個別上書きの例 llm: default: provider: deepseek model: deepseek-chat api_key: sk-xxx overrides: document-analysis: # 重要な文書処理はClaude provider: anthropic model: claude-sonnet-4-6 api_key: sk-ant-xxx image-analyzer: # 画像処理はGPT-4o provider: openai model: gpt-4o api_key: sk-openai-xxx

⚙️ Skill作成・動作

Q SKILL.mdを作ったのにSkillが認識されない
🔴 よくあるエラー
# よくある原因と対処 # 1. ディレクトリ構造が正しいか確認 ls ~/.openclaw/skills/my-skill/ # → SKILL.md が存在するか確認 # 2. ファイル名は正確に SKILL.md（大文字）か # × skill.md, Skill.md, SKILL.txt は認識されない # 3. 再起動してSkillを再読み込み openclaw restart # 4. Skill一覧で認識されているか確認 openclaw skills list
Q Skillが意図しないタイミングで実行される

トリガー条件が広すぎるのが原因です。SKILL.mdのトリガー条件をより具体的にします。

## トリガー条件（良い例と悪い例） # ❌ 悪い例（広すぎる） ## トリガー条件 ユーザーがメッセージを送った時 # ✅ 良い例（具体的） ## トリガー条件 以下のフレーズを含むメッセージのみ実行する: - 「メール整理して」で始まるメッセージ - 「inbox」という単語を含むメッセージ - 毎日09:00のスケジュール実行 以下の場合は実行しない: - 通常の会話・質問 - 他のSkillのトリガー条件と重複する場合
Q Skillが途中で止まる・エラーで終了する
# エラーログを確認 openclaw logs --skill my-skill --tail 50 # よくある原因: # 1. ツールの権限エラー（email_readの認証切れなど） # 2. web_fetch でアクセスできないURLへのアクセス # 3. file_write で書き込み権限がないディレクトリ # 4. レートリミット到達 # SKILL.md にエラーハンドリングを追記する
💡 SKILL.md に追記すると改善：
「エラーが発生した場合は、詳細なエラーメッセージとともにユーザーに報告し、処理を中断してください。自動リトライはしないでください。」
Q 日本語のメッセージに対する応答の精度が低い

SKILL.mdとシステムプロンプトに日本語指定を明示すると大幅に改善します。

## 言語設定（SKILL.md の冒頭に追記） このSkillはすべて日本語で動作します。 - ユーザーへの返答は必ず日本語で行う - 敬語（丁寧語）を使用する - 専門用語は必要に応じて英語を括弧で補足する ## config.yaml でグローバルに設定する場合 system_prompt: | あなたは日本語で動作するアシスタントです。 すべての応答を日本語で行い、自然な敬語を使用してください。
💡 LLM選択も影響： DeepSeekは日本語品質がClaudeより若干低いことがあります。日本語精度を重視する場合はClaudeを試してください。
Q 複数のSkillが競合して意図しない動作をする

トリガー条件が重複しているか、Skillの優先度が設定されていない可能性があります。

# config.yaml でSkillの優先度を設定 skills: priority: - task-manager # 最優先（1番目に評価） - email-sorter - news-summary - catch-all # デフォルト（最後に評価） # SKILL.md で「このSkillが担当しないケース」を明示 ## 実行しない条件 以下のメッセージはこのSkillで処理しない: - 「タスク」を含む → task-manager Skillに委任 - 「メール」を含む → email-sorter Skillに委任

💰 コスト・課金

Q 予想以上のAPI料金が発生した。どうすれば防げたか？
🔴 重大な事故

$1,100超の請求事例はコスト上限未設定が原因でした。以下の設定で必ず防止してください。

# config.yaml の必須コスト設定 limits: monthly_budget: 20 # $20/月 で強制停止 daily_budget: 5 # $5/日 で強制停止 alert_at_percent: 80 # 80%到達でTelegramに警告 # 各LLMプロバイダーでも個別に上限を設定（二重防止） # Anthropicコンソール: Usage Limits → Hard limit を$25に設定 # OpenAIコンソール: Usage limits → Hard limit を$25に設定
⚠️ コスト確認コマンド：
openclaw cost status で現在の使用額を即座に確認できます。週1回は確認する習慣を。
Q コストを抑えながら性能を維持するには？
  • ✔ タスク別モデル使い分け： 定型処理はGPT-4o mini（$0.15/1M）、高品質が必要な文書作成はClaude Sonnet
  • ✔ プロンプトキャッシュを活用： AnthropicのPrompt Caching機能で繰り返しのシステムプロンプトを90%割引
  • ✔ Skillのデバッグはminiモデルで： 新Skillのテストは最安値モデルで。動作確認後に本番モデルに切り替える
  • ✔ スケジュール実行を夜間に： リアルタイム性が不要な処理は深夜に実行して日中のレートリミットを回避
Q 無料でOpenClawを使う方法はある？

完全無料は難しいですが、コストを最小化する方法はあります。

  • ✔ Anthropic： 新規登録で一部の無料クレジットあり（金額・期間は変動）
  • ✔ Zhipu AutoGLM： 学生向け無料プランあり
  • ✔ OpenRouter： 一部のモデルは無料枠あり（制限付き）
  • ✔ Ollama： ローカルLLM（Llama3等）を使えばAPIコストゼロ。ただしGPU必要で性能は劣る

🔐 セキュリティ

Q インストールしたSkillが安全かどうか確認する方法は？
# Skillの安全確認チェックリスト # 1. SKILL.md を必ず読む（ソースコードを確認） cat ~/.openclaw/skills/suspicious-skill/SKILL.md # 2. 使用ツールを確認（不審な権限がないか） # 危険なツールの例: # - system_exec (任意コマンド実行) ← 要注意 # - file_delete (ファイル削除) # - email_send (勝手にメール送信) # - network_request (外部への通信) # 3. GitHubでSkillのコミット履歴を確認 # 最近の更新で何かが変わっていないか # 4. コミュニティの評価・レビューを確認 # OpenClaw公式のSkillレジストリで評価確認
⚠️ ClawHavoc攻撃では「便利そうなSkill」に偽装した悪意あるコードが配布されました。公式リポジトリ外のSkillは特に慎重に。

🟢

Q エージェントが意図しないメールを送信してしまった場合は？
🚨 緊急対処手順
  1. 即座に OpenClaw を停止: openclaw stop
  2. 送信されたメールの内容・送信先をログで確認
  3. 必要に応じて相手先に「誤送信」を連絡
  4. メールの内容に機密情報が含まれていれば情報セキュリティ担当に報告
🛡️ 再発防止：SKILL.mdに必須ルールを追記
## 重要ルール（全Skillに追記を推奨） メールの送信・削除・転送は、必ず事前にユーザーに 送信内容・宛先を提示して承認を得てから実行する。 承認なしに外部へメールを送信することは禁止。

🔮 今後・応用

Q OpenAIにPeter Steinberger氏が移籍した後、OpenClawはどうなる？

2026年2月14日の発表では、OpenClawはオープンソース財団に移管されることが決定されています。

  • ✔ 継続的な開発： オープンソース財団管理のもとコミュニティドリブンの開発が続く
  • ✔ OpenAIとの連携強化： GPTシリーズとの統合がより深まる可能性
  • ✔ 競合の加速： Tencent・ByteDance等の中国テック大手の参入で開発競争が激化
  • ✔ スキルが無駄にならない： SKILL.mdの記述方法は他のエージェントフレームワークにも応用可能
Q OpenClawのスキルアップが、将来どんな仕事に役立つ？
  • ✔ プロンプトエンジニアリング力： SKILL.mdの設計はLLMへの指示設計そのもの。あらゆるAIツールに応用できる
  • ✔ AIシステム設計力： トリガー・ツール・エラーハンドリングの設計はエージェント開発の基礎
  • ✔ 自動化・効率化の実績： 「AIエージェントで〇〇を自動化した」という具体的実績は希少価値が高い
  • ✔ マルチエージェント理解： CrewAI・LangChainへの移行が容易になる。企業のAI導入を主導できる
