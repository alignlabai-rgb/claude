1. API概要

OpenClawはローカルで動作するREST APIサーバーを内蔵しており、外部アプリケーションからプログラム的にエージェントを操作できます。デフォルトではhttp://localhost:3457でリッスンします。

🔑 認証

APIアクセスにはベアラートークンが必要です。初回セットアップ時に自動生成されます。

# ヘッダーに認証トークンを含めてリクエスト
curl -H "Authorization: Bearer YOUR_API_TOKEN" \
     http://localhost:3457/api/v1/status

2. タスク管理API

POST /api/v1/tasks

新しいタスクを作成・送信します。

{
  "message": "明日の会議のアジェンダを作成して",
  "priority": "high",
  "skill_hints": ["calendar-sync", "email-manager"],
  "max_tokens": 4096,
  "callback_url": "https://your-webhook.example.com/callback"
}

レスポンス例：

{
  "task_id": "task_abc123",
  "status": "queued",
  "created_at": "2026-03-21T10:30:00Z",
  "estimated_completion": "2026-03-21T10:31:30Z"
}
GET /api/v1/tasks/{task_id}

タスクのステータスと結果を取得します。

{
  "task_id": "task_abc123",
  "status": "completed",
  "result": "会議アジェンダを作成し、参加者全員にメール送信しました。",
  "tokens_used": 2847,
  "duration_ms": 4520
}
GET /api/v1/tasks

タスク一覧を取得します。ステータスやページネーションでフィルタ可能。

クエリパラメータ：?status=completed&limit=20&offset=0

DELETE /api/v1/tasks/{task_id}

実行中または待機中のタスクをキャンセルします。

3. Skill管理API

GET /api/v1/skills

インストール済みSkillの一覧を取得。

POST /api/v1/skills/install

マーケットプレイスからSkillをインストールします。

{
  "skill_name": "deep-research",
  "version": "latest"
}
PUT /api/v1/skills/{skill_name}/config

Skillの設定を更新します。

4. システム管理API

GET /api/v1/status

システムステータス（稼働時間、メモリ使用量、実行中タスク数など）を取得。

GET /api/v1/usage

API使用量統計（トークン消費量、コスト、タスク数）を取得。

5. WebSocket リアルタイムAPI

🔄 リアルタイム通知

ws://localhost:3457/wsに接続することで、タスク進捗やシステムイベントをリアルタイムで受信できます。

  • task.started： タスク実行開始
  • task.progress： タスク進捗更新
  • task.completed： タスク完了
  • task.failed： タスク失敗
  • system.alert： システムアラート（コスト上限接近等）

6. レート制限

  • デフォルト制限： 60リクエスト/分（設定で変更可能）
  • タスク送信： 10リクエスト/分
  • WebSocket接続： 同時5接続まで
