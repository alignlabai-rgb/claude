Base44 Skills
エージェントのハリュシネーションを構造で止める

AI コーディングエージェント（Claude Code / Cursor / OpenClaw など）がプラットフォーム固有の操作を誤らないよう、 「Skills」という構造化された命令セットの仕様を定めたオープンスタンダード。 エージェントに渡す「地図」を統一することで、幻覚（hallucination）によるAPI誤呼び出しを根本から防ぐ。

🧠 なぜエージェントはハリュシネーションを起こすのか

Claude Code や Cursor などのエージェントは、学習データに基づいて SDK メソッドを「推測」する。 プラットフォームが頻繁にAPIを変更すると、エージェントは古い・存在しないメソッドを自信満々に呼び出してしまう。 Base44 Skills はその「推測」をなくし、エージェントに確実な地図を渡す仕組み。

3つのコアSkills

エージェントが Base44 プラットフォームを誤りなく操作するために設計された、3種類の公式 Skill：

base44-cli

プロジェクト初期化、スキーマ管理、リソース設定、デプロイメントを担当。

  • プロジェクト構造の作成
  • DBスキーマ定義・マイグレーション
  • 本番へのデプロイ操作
  • 環境変数・設定の管理
base44-sdk

JavaScript SDK を使った機能実装をガイド。エージェントが正しいメソッドを使えるよう厳密に定義。

  • 認証（Auth）の実装パターン
  • DB クエリの正しい書き方
  • バックエンド関数の呼び出し
  • 型定義・バリデーション
base44-troubleshooter

本番障害のデバッグをエージェントが支援。ログ解析・エラートレース読解を構造化。

  • エラーログの解析手順
  • スタックトレースの読み方
  • よくある失敗パターン集
  • 修正の優先順位ガイド

Skills の仕組み — SKILL.md

各 Skill は SKILL.md というマークダウンファイルで定義される。エージェントはこのファイルを読み込んで、プラットフォーム固有の操作方法を「確実な事実」として保持する。

  • メタデータ Skill の名前・バージョン・対象プラットフォーム
  • 命令セット 正しい SDK メソッド・コマンド・ワークフロー
  • 禁止事項 やってはいけないパターン（Anti-patterns）
  • 例示 正しい実装例・間違い例のペア

💡 ポータビリティ： SKILL.md はオープン標準のため、Claude Code・Cursor・Anti-Gravity など対応エージェントならどれでも同じ Skill を使い回せる。

SKILL.md の例（抜粋）：

# base44-sdk Skill
version: 1.2.0
platform: base44

## 正しい認証の実装
```js
// ✅ 正しい
const user = await App.auth.me();

// ❌ 間違い（ハリュシネーション例）
const user = await base44.getUser();  // 存在しない
```

## DBクエリ
```js
// ✅ 正しい
const items = await App.db
  .collection("tasks")
  .filter({ status: "active" })
  .list();
```

インストールと使い方

グローバルインストール（推奨）

一度インストールすれば、接続するどのエージェントにも自動で Skill が引き継がれる：

npx skills add base44/skills -g

これにより Claude Code・Cursor などが Base44 プロジェクトで操作を行う際、Skills の定義に沿った正確なコードを生成する。

プロジェクト単位での追加

特定のプロジェクトのみに適用する場合：

cd my-base44-project
npx skills add base44/skills
# → .skills/ ディレクトリが生成される

OpenClaw エコシステムとの関係

Base44 Skills は、OpenClaw / PicoClaw / NanoClaw と同じ「Skills エコシステム」の上に乗る概念。各 Claw スタックが Base44 Skills を読み込めば、Base44 プラットフォームへの操作を自律的に行えるようになる。

スタックBase44 Skills の活用場面
PicoClawローカルからBase44プロジェクトを操作するCLIエージェント
NanoClawCI/CDパイプラインでBase44への自動デプロイ
OpenClaw本番規模のBase44アプリを監視・自律運用するオーケストレーター

AI Creator Hub での活用イメージ

  • コンテンツ評価結果を Base44 上のダッシュボードアプリに自動反映
  • OpenClaw から Base44 の管理画面を Skills 経由で操作
  • PicoClaw の HEARTBEAT で Base44 のデプロイ状態を定期チェック
  • Skills 定義をそのままチームの非エンジニアとも共有

💡 Base44 への変更が起きても、SKILL.md を更新するだけでエコシステム全体のエージェントが最新の正しい操作方法を参照できる。

📌 名前の混同について

「Base44」と検索すると、Wix が 2025年6月に $80M で買収したノーコードアプリ生成サービス「Base44」が検索上位に出てくるが、これは同名の別サービス。このページで扱うのはエージェントの Skills 標準・OpenClaw エコシステムとの連携文脈における Base44 Skills の概念。プラットフォームとしての Base44（Wix）を使う場合でも、Skills 標準は同様に適用される。
