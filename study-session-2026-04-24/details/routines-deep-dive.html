AIが勝手に仕事を始める
Routinesとは何か

「呼んだら動く」から「設定したら永遠に動き続ける」へ。チームが画面の前にいなくても、AIが仕事を進めている世界がすでに始まっている。

リリース: 2026年4月14日
種別: Research Preview
前提知識: 不要
パラダイムシフト
何がどう変わるのか
チャット型AIとRoutinesの本質的な違いは「誰が仕事を始めるか」だ。
❌ チャット型AI（今の使い方）
  • 毎朝「今日のM&Aニュースをまとめて」と手動で入力する
  • 画面の前に座っているときしかAIは動かない
  • 土日・深夜・出張中はAIが止まる
  • 同じ作業を毎日・毎週繰り返し頼み続ける必要がある
  • 「指示する手間」がゼロにならない限り、効率化に上限がある
✅ Routines（設定後の世界）
  • 一度設定すれば、毎週月曜8時に自動でレポートが届く
  • 人間がオフラインでも、クラウド上でAIが仕事を続ける
  • 土日・深夜も、海外出張中も、AIは止まらない
  • 「繰り返し頼む手間」が永遠にゼロになる
  • 人間の仕事は「設定」と「判断」だけになる

Routinesの本質 — AIとの関係が「同期」から「非同期」に変わる

チャット型AIは「同期型」— 人間とAIが同時にオンラインで対話する電話のようなもの。Routinesは「非同期型」— 人間が設定して離れても、AIがバックグラウンドで仕事を続けるメールのようなもの。

電話型（チャット型AI）
  • かけている間しか会話できない
  • 切ったら止まる
  • 毎回かけ直す必要がある
  • 相手（AI）も自分も同時に時間を使う
メール型（Routines）
  • 送信したら自分は次の仕事ができる
  • 返信が来たら確認する（非同期）
  • 定期メールの設定は一度だけ
  • 自分がオフラインでも仕事が進む
💡 ゲームチェンジの意味

これまでAIは「人間の集中力と時間」を節約するツールだった。Routinesは「人間の存在そのもの」を作業から切り離す。寝ている間に仕事が完成しているという体験が、初めて現実になった。

Routinesの仕組み
AIを動かす3つのきっかけ
Routinesは3種類の「トリガー（引き金）」で起動する。MAチームの業務では主にスケジュール型が即効性が高い。
🕐
Scheduled — 時間スケジュール
「毎週月曜8時に」自動で動く
カレンダーのように時間を指定するだけ。毎日・毎週・毎月、好きな周期で繰り返し実行できる。
MAチームでの活用例
毎週月曜8時：先週のM&Aニュースをスキャン → SBIグループ関連を抽出 → 要約レポートを生成。出社したらレポートが届いている
今すぐ使える最高の即効性
⚡
API — 外部からの合図
「新案件が登録されたら」自動で動く
社内システムや外部サービスからの信号で即座に起動する。「何かが起きたら」AIが自動で対応する仕組み。
MAチームでの活用例
案件管理システムに新案件が登録 → 自動でClaude Codeが起動 → ターゲット企業の初期DD調査を開始 → 担当者に通知

🗓 週次M&Aニュースレポート（今すぐ実装可能）

毎週月曜8時に自動起動。Bloomberg・Reuters・日経を横断スキャンして、SBIグループ関連のM&Aニュースを抽出・要約する。

Routineの設定内容（イメージ）
毎週月曜 08:00 に実行： 先週7日間のM&A関連ニュースを収集。 SBIグループの事業領域（金融・証券・保険・フィンテック）に関連する案件を優先的に抽出。 各案件を「概要・関係者・SBIへの影響度（高/中/低）」でまとめる。 日本語のWeeklyレポートとして出力する。
何が変わるか

月曜朝に「先週のニュース調べて」と入力する作業が永遠になくなる。出社したらレポートが届いている。年間50週 × 15分 = 約12時間の解放。

🌏 ターゲット国の規制モニタリング（応用）

投資先・検討中の国の規制動向・法改正情報を自動モニタリング。担当者が気づく前にAIが情報を収集している。

Routineの設定内容（イメージ）
毎週水曜 07:30 に実行： ベトナム・インドネシア・インドの金融規制・外資規制に関する最新情報を収集。 前週から変更があった項目のみ抽出し、日本語でサマリーを作成。 「対応が必要な変更」には⚠️マークを付ける。
💡 重要な変化

担当者が「調べようと思っていた」のに手が回らなかった情報収集を、AIが代わりに続けてくれる。情報の漏れがなくなる。

発展的な活用
Routines × サブエージェントを組み合わせると
Routinesは「いつ動くか」を決める。サブエージェントは「何人で動くか」を決める。組み合わせると「週1回、5人のAIチームが自律的にリサーチを完了させる」が実現する。

Routinesの導入ステップ（コード不要）
Claude Code の Routines 機能を使う前の段階でも、claude.ai の定期利用習慣から始められる。

Step 1 — 今週中

claude.aiで始める
  • 毎朝同じプロンプトを「定型文」として保存
  • 月曜朝に必ず「週次M&Aニュース収集」を実行する習慣をつける
  • どんな情報をどんなフォーマットで欲しいかを練る
目的

「何を自動化したいか」を明確にする。Routineの設計は習慣化から始まる。

Step 2 — 1ヶ月以内

Claude Code Routines を設定
  • Claude Code（Max以上のプラン）でRoutines機能を有効化
  • 最初の1つ：週次M&Aニュースレポートを設定
  • 月曜朝に自動で届くことを確認
  • フォーマットを調整して精度を上げる
最初の感動ポイント

月曜朝、自分が何もしていないのにレポートが届いている体験を一度すれば、もう元には戻れない。

Step 3 — 3ヶ月後

Routinesを複数展開
  • 週次ニュース・規制モニタリング・競合動向を3本のRoutineで並列運用
  • サブエージェントと組み合わせて精度・スピードを向上
  • 「定期的な手動作業」がほぼゼロになった状態
  • 人間の時間を「収集・集計」から「判断・実行」に全振り
3ヶ月後のチームの姿

情報収集は全自動。チームの全集中力が「どう動くか」の意思決定に使われている。
