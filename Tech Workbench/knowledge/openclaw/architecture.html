The Recursive Architecture

Understanding the OODA Loop

OpenClaw (Official OpenAI Operator) は、単なる「チャットボット」ではありません。
それは、目標が達成されるまで無限に自己対話を繰り返す再帰的ループ・プログラムです。

The OODA Loop

軍事戦略で用いられる OODA (Observe-Orient-Decide-Act) ループが、そのまま実装されています。

┌─────────────────────────────────────────────────────────────┐ │ OBJECTIVE │ │ "このリポジトリの全テストを通るように修正せよ" │ └───────────────────────────┬─────────────────────────────────┘ │ ┌───────▼───────┐ ┌───────────────┤ 1. OBSERVE │ (環境認識) │ │ 「現在のエラーログを見る」 │ └───────┬───────┘ │ │ │ ┌───────▼───────┐ │ │ 2. ORIENT │ (状況判断) │ │ 「これは非同期処理のタイミング問題だ」 │ └───────┬───────┘ │ │ ┌──────┴───────┐ ┌───────▼───────┐ │ 4. ACT │◄──────┤ 3. DECIDE │ (意思決定) │ ツール実行 │ │ 「wait時間を増やしてリトライしよう」 └──────────────┘ └───────────────┘ │ ▼ (New State / New Logs) ──► Loop Back to OBSERVE

1. Observe (観測)

エージェントは盲目です。ls, cat, browser.screenshot などのツールを使わない限り、世界が見えません。
OpenClawは最初のステップで必ず「現状分析」のためのコマンドを乱射します。

2. Orient (方向付け/思考)

観測結果（ツール出力）を元に、内部で推論（Reasoning）を行います。
「ファイルAを修正したのにエラーが変わらない...ということは、原因はファイルBにあるのでは？」といった仮説構築フェーズです。

3. Decide (決定)

仮説に基づき、次に行うべき「たった一つの行動」を決定します。
「ファイルを全部書き換える」といった無謀な計画ではなく、検証可能な最小の一手を打ちます。

4. Act (行動)

実際にファイルシステムやブラウザに対し、副作用のある操作（Write/Click）を実行します。
この結果が、次のループの「観測」対象となります。

State Management

この無限ループを支えているのが "Scratchpad" (思考ノート) です。
エージェントは各ステップごとに、自分の思考過程を隠しファイル（またはメモリ上）に追記していきます。

Step 15: - 試したこと: src/utils.ts の修正 - 結果: 失敗 (Error: Timeout) - 次の仮説: タイムアウト設定が短すぎる - 次の行動: .env の TIMEOUT 値を確認する

これにより、Operator（あなた）は後から「彼がなぜその修正をしたのか」を完全に追跡（Audit）できるのです。
