ENVIRONMENT / VERSION CONTROL

Git (Time Control)

開発における「セーブポイント」作成ツール。失敗しても過去に戻れる安心感が、大胆な実験（Vibe Coding）を可能にします。

← Workbenchへ戻る

基本の3ステップ

git add .

「ステージング」：変更したファイルを「次のセーブ対象」として選ぶ。

git commit -m "機能追加"

「コミット」：セーブを実行する。メッセージは何をしたかのメモ。

git push

「プッシュ」：手元のセーブデータをクラウド（GitHub等）にアップロードする。

AIと使う Git

難解なコマンド（rebase, cherry-pick等）を覚える必要はありません。複雑な操作が必要なときは、Claude Code や Cursor にこう頼んでください。

「直前のコミットを取り消して、変更内容はそのまま残しておいて（Soft Reset）」

AIは正確なコマンドを知っています。あなたは「時間をどう操作したいか」を指示するだけです。

困ったときの呪文

git status

現在の状況（変更されたファイル、ステージングの状態）を確認する。迷ったらまずはこれ。

git log --oneline

過去のセーブポイント（コミット履歴）を一覧表示する。
