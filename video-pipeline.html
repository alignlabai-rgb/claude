システム内での役割

パイプラインの
コアエンジン

動画制作モジュールはAI Creator Hubのメイン収益エンジンです。 ネタ生成AIからテキストを受け取り、完成動画ファイルを評価システムへ渡す 中核パイプラインとして機能します。全ステップはOpenClawが自律制御します。

  • 入力元: ネタ生成LLM（テキスト・JSON）
  • 出力先: 評価システム（動画ファイル + メタデータ）
  • 処理時間: ショート動画 約8〜15分 / 長尺動画 約45〜90分
  • 並列処理: 最大4本同時生成（GPU依存）
  • 失敗時: OpenClawが自動リトライ、3回失敗で人間へアラート

モジュール入出力定義

# OpenClaw スキル定義 (SKILL.md 抜粋) name: video-pipeline version: "2.1.0" input: topic_text: string # ネタ文章 genre: string # shorts|long|podcast target_sec: int # 目標尺（秒） style_preset: string output: video_path: string # 完成動画パス thumbnail_paths: list metadata: object # タイトル/説明/タグ cost_usd: float # API使用コスト
制作フロー

7ステップで完成動画へ

1

ネタ文章の受け取り & 前処理

別ラインから生成されたテキストを受け取り、ジャンル・ターゲット・トーンを自動分析。不適切コンテンツフィルタリングも同時実行。

入力: テキスト GPT-5.4 / Claude Sonnet 4.6 / Gemini 3.1 Pro
2

動画スクリプト生成

ネタ文章をYouTube/ショート動画向けのスクリプトに変換。フック（冒頭掴み）・本編構成・CTA（行動喚起）を自動設計。字幕テキストも同時生成。

// スクリプト生成プロンプト例 const script = await llm.generate({ system: "あなたはYouTube動画脚本家です", prompt: `以下のネタを元に動画スクリプトを作成: - フック: 最初の3秒で視聴者を掴む - 本編: ${targetDuration}秒の構成 - CTA: チャンネル登録・いいね誘導 ネタ: ${sourceText}` });
3

ビジュアル素材生成

スクリプトの各シーンに対応する画像・映像をAIで生成。スタイルの一貫性を保ちながら、シーンごとに最適な構図・色調を自動選択。

Midjourney API DALL-E 3 RunwayML Gen-3 Kling AI
4

音声合成（ナレーション）

スクリプトをリアルな日本語音声に変換。複数の声質から自動選択し、感情・速度・イントネーションを最適化。BGM選択も自動。

ElevenLabs VOICEVOX CoeFont Suno AI (BGM)
5

字幕・テロップ生成

音声に同期した字幕を自動生成。強調ワードのハイライト・エフェクト・フォントスタイルも動画の雰囲気に合わせて自動調整。

Whisper (文字起こし) FFmpeg (字幕埋め込み)
6

動画編集・結合

映像・音声・字幕をFFmpegで自動合成。トランジション・カット編集・テロップアニメーションをルールベースで適用し、完成動画を生成。

# FFmpegによる自動合成例 ffmpeg -i video_scenes.mp4 \ -i narration.mp3 \ -i bgm.mp3 \ -filter_complex "[1:a]volume=1.0[v1]; [2:a]volume=0.15[v2]; [v1][v2]amix=inputs=2[aout]" \ -map 0:v -map "[aout]" \ -vf "subtitles=subs.srt" \ output_final.mp4
7

サムネイル生成

動画の内容から最も視聴クリックされやすいサムネイルを複数パターン生成。A/Bテスト用に3〜5パターンを自動作成し評価システムへ送付。

Canva API DALL-E 3 出力: 評価システムへ

他モジュールとの
データフロー設計

動画パイプラインはOpenClaw Skillとして実装され、前後のモジュールとメッセージキューで非同期連携します。 各ステップの進捗はOpenClawダッシュボードでリアルタイム確認できます。

# OpenClaw ワークフロー定義 (抜粋) workflow: video-creation steps: - name: script-gen skill: llm-writer on_error: retry(3) - name: visual-gen skill: image-pipeline parallel: true # 音声合成と並列実行 - name: voice-gen skill: elevenlabs-tts parallel: true - name: compose skill: ffmpeg-composer depends_on: [visual-gen, voice-gen] - name: evaluate skill: content-evaluator # → 評価システムへ

インフラ要件

FFmpeg合成サーバー
CPU: 8コア以上 / RAM: 16GB以上 / SSD: 500GB（一時ファイル保存用）。クラウドならGCP n2-standard-8が適合。
並列4本処理時のボトルネック
最大4本並列はRunwayML APIの同時リクエスト上限（Proプラン）による制限。追加クレジットで8本並列も可能。
ストレージ設計
生成中ファイルはローカルSSD → 完成後はS3/GCSへ移動。評価通過後のみ長期保存。未通過は48時間後に自動削除。
