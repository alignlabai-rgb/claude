Physics of the AI Mind: Context, Caching & Latency

1. The Anatomy of a Prompt

あなたが $ claude "fix bug" と打った瞬間、実際にAPIに送信されるのは「たった2単語」ではありません。
CLIはバックグラウンドで巨大なテキストの塊（Prompt Assembly）を構築しています。

[ SYSTEM PROMPT (Fixed) ] ─────────────────────────────┐ │ "You are Claude Code, a sophisticated CLI agent..." │ │ "Current Date: 2026-02-11" │ │ "OS: Windows 11 / Shell: PowerShell" │ └──────────────────────────────────────────────────────┘ ▼ [ LONG-TERM MEMORY (Dynamic) ] ────────────────────────┐ │ Project Context: "Auth Module Refactoring..." │ │ User Rules: "Always use TypeScript explicit types" │ └──────────────────────────────────────────────────────┘ ▼ [ CONVERSATION HISTORY ] ──────────────────────────────┐ │ User: "List files" │ │ Asst: [ToolUse: ls] │ │ Tool: "src, test, package.json..." │ └──────────────────────────────────────────────────────┘ ▼ [ TARGET FILES (The Heaviest Part) ] ──────────────────┐ │ src/app.ts (Content...) │ │ src/utils.ts (Content...) │ └──────────────────────────────────────────────────────┘ ▼ [ NEW INSTRUCTION ] ───────────────────────────────────┐ │ User: "fix bug" │ └──────────────────────────────────────────────────────┘

Impact: ファイルを読み込めば読み込むほど、[ TARGET FILES ] の領域が肥大化し、レスポンス速度（TTFT）が低下し、コストが指数関数的に増大します。

2. Prompt Caching (The Speed Hack)

Claude Codeの爆速レスポンスを支えているのが Anthropic API Prompt Caching です。
送信するプロンプトの「前半部分」が変わらなければ、API側でキャッシュされたメモリを再利用します。

Cache Breakpoint Strategy
キャッシュを有効にするには「変化しないもの」を先に、「変化するもの」を後に送る必要があります。
Claude Code CLIは、以下のように自動構成しています：
  1. Static System Prompt (Cached: 100%)
  2. Project Spec (.ai/CONTEXT.md) (Cached: High)
  3. Read Files (Cached: Medium - 変更されるとキャッシュ切れ)
  4. New Query (No Cache)

※ 頻繁に書き換わるファイル（ログファイル等）を読みっぱなしにすると、キャッシュヒット率が下がり、動作が重くなります。

3. Adaptive Thinking vs Latency

Opus 4.6 の「思考」はタダではありません。内部的には「見えないトークン」を生成しています。

Effort Level Hidden Tokens (Thinking) Latency (待ち時間) Use Case
Low (Sonnet-like) ~500 tokens 1-3 sec 単純なバグ修正、Lintエラー対応
Medium (Default) 2,000 - 5,000 tokens 5-15 sec 通常の実装、要件定義
High (Deep Thought) 10,000+ tokens 30-60 sec アーキテクチャ設計、複雑なリファクタリング

$ claude --effort low "fix typo" のように使い分けることで、無駄な待ち時間とAPIコストを削減できます。
