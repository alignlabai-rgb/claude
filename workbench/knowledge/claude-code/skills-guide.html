Claude Code Skills

「毎回同じことを説明する」から「一度定義したら誰でも使える」へ。Skillsは再利用可能なAI能力をファイルとして標準化し、チーム全体で共有するオープン規格。属人化を解消し、MAチームのDD能力を組織の資産にする。

規格: オープン標準
対応: Claude Code / Agent SDK
場所: .claude/agents/ または Skills Hub
用途: チーム標準化 / DD自動化
基本概念
Skillsとは何か — 「プロンプト」から「能力ファイル」へ
チャット型AIの最大の弱点は「記憶がない」こと。毎回同じ説明から始まる。Skillsはその問題を根本から解決する。
❌ 旧来のチャット型（毎回リセット）
  • 「あなたはM&A財務アナリストです。EBITDAを計算し…」を毎回冒頭に書く
  • 同じ手順・判断基準・出力フォーマットをメンバーごとに再発明
  • ベテランが退職すると知識も消える（属人化）
  • 品質が担当者のプロンプト力に依存する
  • 改善しても次のセッションには反映されない
✅ Skills（能力の資産化）
  • Skill定義ファイルに「役割・手順・ツール・出力形式」を一度書く
  • チーム全員が同じSkillを呼び出す（標準化）
  • Skillは組織のGitリポジトリで管理・バージョン管理
  • 改善はSkillファイルを更新するだけで全員に反映
  • 新メンバーが即日プロレベルのDDを実行できる
本質的な変化

Skillsは「AIへの指示」ではなく「組織の能力定義」。ベテランのDDノウハウをファイルに書き出すことで、その知識が永続的な組織資産になる。これが「属人化解消」の実体。

Skill定義ファイル
Skill定義ファイルの書き方
Skillは .claude/agents/ 以下に置くマークダウンファイル。YAMLフロントマターで設定、本文でロールと手順を定義する。
# .claude/agents/dd-financial-analyst.md
---
name: dd-financial-analyst
description: |
  M&A財務デューデリジェンスの専門Skill。
  財務諸表（B/S・P&L・CF）の分析、EBITDA/FCFの算出、
  バリュエーション、リスクフラグの特定を実施する。
  対象企業の財務資料が提供されたときに使用する。
tools:
  - Read        # PDFや財務資料ファイルの読み込み
  - Write       # DDレポートの生成
  - WebSearch   # 公開財務データ・業界ベンチマークの検索
model: claude-sonnet-4-6   # デフォルトモデル（高コスト分析時はOpus 4.6を指定）
---

# 財務DDエージェント

## 役割
あなたはM&A専門の財務デューデリジェンスアナリストです。
提供された財務資料を分析し、投資委員会が意思決定に使えるレベルの
財務DDレポートを作成します。

## 分析フレームワーク

### 1. 財務実績の分析
- 過去3〜5年の売上高・EBITDA・純利益の推移とCAGR
- P&LとCFの乖離（利益の質の評価）
- 運転資本の変動とその意味

### 2. バリュエーション
- EV/EBITDA倍率（業界比較）
- DCF（WACC・ターミナル成長率は業界標準を適用）
- トランザクションコンパラブル

### 3. リスクフラグ（必ず明示）
- のれん・無形資産の急増
- 一時項目の常態化
- 関連当事者取引
- 偶発債務・簿外負債

## 出力形式
エグゼクティブサマリー（1ページ）→ 詳細分析 → リスクマトリクス の順で出力。
数値は全て円換算・百万円単位。信頼区間が低い推計値には★マークを付ける。

YAMLフロントマターの主要フィールド

フィールド説明必須
nameSkill識別子（@メンションで呼び出す名前）✅
descriptionいつ使うか・何をするかの説明（AIが判断に使う）✅
toolsこのSkillに許可するツールのリスト推奨
model使用モデルの指定（省略時は親から継承）任意
max_turns最大ターン数の制限任意

Skillの呼び出し方

# 方法1: @メンションで直接指定
"@dd-financial-analyst 対象企業XのDDを開始してください"

# 方法2: Claude Codeが自動判断
"財務DDを実施してください"
# → descriptionを読んでdd-financial-analystを自動選択

# 方法3: Agent TeamsでOrchestrator経由
"DD Swarmを起動してください"
# → Orchestratorが3つのSkillを並列起動
Descriptionが重要

descriptionはAIが「このSkillをいつ使うか」を判断する根拠。「〜の場合に使用する」「〜が提供されたとき」と使い所を明示すること。

MAチーム実装
MAチーム向けSkillライブラリの設計
DDの4領域（財務・法務・ビジネス・統合）に対応する4つのコアSkillと補助Skillで構成する。
💹 Core Skill 1

dd-financial-analyst

財務諸表分析・バリュエーション・FCF評価。EBITDAマルチプル・DCF・リスクフラグの特定。

  • ツール: Read, Write, WebSearch
  • モデル: Sonnet 4.6（通常）/ Opus 4.6（複雑案件）
  • 出力: 財務DDレポート + バリュエーションレンジ
⚖️ Core Skill 2

dd-legal-reviewer

契約書・定款・コンプライアンス資料の審査。重要条項の抽出・リスク評価・Change of Control条項特定。

  • ツール: Read, Write
  • モデル: Opus 4.6（法的判断精度優先）
  • 出力: 法務リスクマトリクス + 重要条項一覧
📊 Core Skill 3

dd-business-analyst

市場分析・競合調査・シナジー評価・PMI計画。業界ポジション・成長ドライバー・リスク要因の特定。

  • ツール: Read, Write, WebSearch
  • モデル: Opus 4.6（戦略判断）
  • 出力: ビジネスDDレポート + シナジー試算
📋 Synthesis Skill

dd-report-synthesizer

3つのDDレポートを統合してエグゼクティブサマリーを生成。投資委員会向けの意思決定資料を作成。

  • ツール: Read, Write
  • モデル: Opus 4.6（統合・要約）
  • 出力: 投資委員会プレゼン用サマリー

🔍 Utility: market-researcher

特定企業・業界のWebリサーチ。財務・ビジネスDDの補助として呼び出し可能。

📑 Utility: document-extractor

PDFやWordからの情報抽出・構造化。大量資料の前処理に特化。

📧 Utility: communication-drafter

対象企業・アドバイザー向けのメール・質問状の下書き生成。

断絶① 能力の資産化

チャット型AIは「使い捨て」。Skillsは組織の能力をファイル化して永続的に蓄積する。ベテランの知識が組織資産になる初めての仕組み。

断絶② 属人化の解消

優秀なプロンプトエンジニアがいなくても、Skillライブラリが標準品質を保証する。1人のMDが設計したDDフレームワークをチーム全員が再現できる。

断絶③ 1人で1000社へ

Skillsで標準化したDDをRoutinesで自動起動・Agent Teamsで並列実行すると、1人のアナリストが100社を同時監視するシステムが完成する。

DBS銀行との接続

シンガポールDBS銀行が「AIと協働するバンカー」モデルで生産性を倍増させた背景には、こうした能力の標準化がある。Skillsはその仕組みを自社で構築するための技術基盤。SBI新生銀行のMAチームでも、まず財務DDのSkillライブラリを整備することが最初のステップになる。

導入手順
MAチームへのSkills導入ステップ
最初の1週間で動くSkillライブラリを作る最短ルート。
1

既存DDチェックリストを棚卸し（1日）

現在チームが使っているDDフレームワーク・チェックリスト・フォーマットを集める。これがSkillの原材料。

2

財務DDSkillを最初に作る（2日）

dd-financial-analyst.mdを作成。既存チェックリストをそのままskill本文に変換するだけでよい。完璧を求めない。

3

実際のDDで試用・改善（1週間）

リアル案件で使いながら「この手順が足りない」「この出力形式を変えたい」を反映。Skillは使いながら育てる。

4

法務・ビジネスSkillに拡張（2週目）

財務Skillが安定したら法務・ビジネスへ展開。3つ揃えばDD Swarmが組める。

5

Routinesと組み合わせて自動化（1ヶ月後）

毎朝7時に自動でSkillを起動し、ターゲットリスト企業の動向をスキャン。「気づいたら分析が終わっている」状態へ。
