1. 日次メールトリアージ
🟢 初級 Claude Haiku or GPT-4o mini ⏰ 毎朝9:00自動実行
受信トレイを自動で整理し、重要メールだけを朝に通知する
# スキル名: 日次メールトリアージ ## 概要 毎朝9:00に受信トレイを確認し、重要度別に分類して Telegramに日次サマリーを送信する。 ## スケジュール - 毎日 09:00 に自動実行 - 「メール確認」「inbox」と送ると即時実行 ## 使用ツール - email_read: メールの読み取り（送信はしない） - message_send: Telegramへの通知送信 ## 実行手順 1. 未読メール一覧を取得する 2. 各メールを以下の優先度で分類: - 🔴 要対応（24時間以内の返信が必要なもの） - 🟡 要確認（今週中に確認が必要なもの） - 🟢 情報共有（読むだけでOKなもの） - ⚫ スパム・ニュースレター 3. スパムはアーカイブし、他は優先度タグを付与 4. 以下のフォーマットでサマリーをTelegramに送信 ## 出力フォーマット 📬 **メールサマリー（{日付}）** 未読{N}件 🔴 **要対応** ({N}件) - {送信者}: {件名} — {20文字以内の内容要約} 🟡 **要確認** ({N}件) - {送信者}: {件名} 🟢 **情報共有** ({N}件) - {送信者}: {件名} ⚫ スパム{N}件をアーカイブしました ## 注意事項 - メールの送信・削除は行わない。読み取りと分類のみ - 個人情報・機密情報が含まれるメールは要約を省略し 「[要確認]機密情報を含む可能性があります」と表示
💡 カスタマイズポイント： 「🔴要対応」の判定条件を変えると精度が上がります。例：「上司・重要顧客からのメール」「返信が必要な動詞が含まれるメール」など。

2. 会議スケジューラー
🟢 初級 GPT-4o mini ⏰ トリガー実行
「〇〇さんと来週1時間打ち合わせを入れて」で自動調整
# スキル名: 会議スケジューラー ## 概要 自然言語で会議の調整依頼を受け付け、カレンダーを確認して 空き時間を提案し、承認を得てから予定を登録する。 ## トリガー条件 以下のパターンを含むメッセージ: - 「〜と打ち合わせ」「〜と会議」「ミーティングを設定」 - 「〜さんとスケジュール調整」「空き時間を確認」 ## 実行手順 1. メッセージから以下を抽出: - 相手の名前・メールアドレス - 希望日程・時間帯・所要時間 - 会議の目的（任意） 2. 自分のカレンダーで空き時間を3候補取得 3. 候補をユーザーに提示して選択を求める（自動送信しない） 4. ユーザーが承認したら相手にメールで候補を送信 5. 相手が確定したらカレンダーに登録し通知 ## 重要ルール - ステップ3の提示後、必ずユーザーの承認を待つこと - 相手へのメール送信前に必ず文面をユーザーに確認する - 既存の予定を変更・削除しない

[{見出し}]({URL}) {3行要約} 💹 **金融・マーケット** 2. [{見出し}]({URL}) {3行要約} ━━━━━━━━━━━━━━ ⏭️ 次回配信: {次の配信時刻}

4. 為替・株価アラート
🟡 中級 GPT-4o mini ⏰ 1時間ごとに監視
設定したレートを超えたら即アラート。週次サマリーも自動生成
# スキル名: 為替・株価アラート ## 概要 設定した金融指標を定期監視し、閾値を超えたらアラート。 週次でマーケットサマリーを生成する。 ## スケジュール - 平日9:00〜18:00 の間、1時間ごとに監視 - 毎週金曜 18:30 に週次サマリーを送信 ## 監視対象（カスタマイズ可） - USD/JPY: 150.00円を超えたらアラート（上限） - USD/JPY: 140.00円を下回ったらアラート（下限） - 日経225: 前日比±2%でアラート - BTC/JPY: 前日比±5%でアラート ## データソース - web_search でリアルタイムレートを取得 （例: "USD JPY 現在レート site:xe.com"） ## アラートフォーマット ⚠️ **為替アラート** USD/JPY が **{レート}** に到達しました （設定閾値: {閾値}） 変動率: 前日比 {変動率}% 時刻: {時刻} ## 週次サマリーフォーマット 📊 **週次マーケットサマリー（{週}）** | 指標 | 週初 | 週末 | 変動 | |------|------|------|------| | USD/JPY | {値} | {値} | {変動} | | 日経225 | {値} | {値} | {変動} | 今週の注目イベント: {要約} 来週の注目点: {要約} ## 注意 - データ取得はウェブ検索のみ（証券APIへの直接アクセスなし） - 投資判断を促す表現は使わない

5. 学習メモ自動整理（Obsidian連携）
🟢 初級 Claude Haiku ⏰ トリガー実行
「メモ：〇〇」と送るだけで自動分類・タグ付けしてMarkdownに保存
# スキル名: 学習メモ整理 ## 概要 メッセージで送られた学習メモを自動的に分類・タグ付けして Markdownファイルに整理保存する。Obsidian形式に対応。 ## トリガー条件 - 「メモ：」または「memo:」で始まるメッセージ - 「保存：」「note:」「覚書：」で始まるメッセージ ## 実行手順 1. メッセージからメモ内容を抽出 2. 内容を分析して以下を自動判定: - カテゴリ: AI/テック/金融/語学/その他 - タグ: 内容から3〜5個のキーワードタグ - 重要度: 高/中/低 3. 以下のフォーマットで ~/notes/YYYYMM.md に追記 4. 完了メッセージを返信 ## 保存フォーマット（Obsidian対応） --- date: {YYYY-MM-DD HH:mm} category: {カテゴリ} tags: [{タグ1}, {タグ2}, {タグ3}] importance: {高/中/低} --- ## {メモタイトル（内容から自動生成）} {メモ本文} **AIによる補足**: {関連する背景知識・補足情報を1〜2文で追加} --- ## 保存先 ~/notes/memos/{YYYYMM}/notes.md （月別に自動分類）
💡 発展形： 週1回「今月のメモを振り返って、学習の弱点を分析して」とメッセージするだけで学習レポートを自動生成できます。

気になる商品の価格を監視して最安値になったら即通知
# スキル名: 価格追跡アラート ## 概要 指定した商品の価格をAmazon・楽天等で定期監視し、 目標価格に達したらアラートを送信する。 ## トリガー条件（登録） 「追跡: {商品名} 目標価格: {金額}円」のメッセージで登録 ## スケジュール（監視） 毎日 09:00 と 21:00 に登録商品を自動チェック ## 実行手順 1. 登録リスト（~/tracking/products.json）を読み込み 2. 各商品をウェブ検索で現在価格を取得 - Amazon, 楽天, ヨドバシ, ビックカメラ等 3. 目標価格以下の場合はアラートを送信 4. 価格履歴を~/tracking/history.json に記録 ## 登録フォーマット（products.json） { "products": [ { "name": "Sony WH-1000XM5", "target_price": 35000, "currency": "JPY", "registered": "2026-03-14" } ] } ## アラートフォーマット 🎉 **価格アラート！** 「{商品名}」が目標価格に到達しました 現在価格: **{価格}円**（目標: {目標}円） 販売店: {店舗名} リンク: {URL} 前回確認時: {前回価格}円（{変動率}% 下落）

8. 深掘りリサーチ（マルチソース調査）
🟡 中級 Claude Sonnet ⏰ トリガー実行
「〇〇を調べてレポートにまとめて」で複数ソースから包括的なレポート作成
# スキル名: 深掘りリサーチ ## 概要 指定されたトピックを複数の情報ソースから調査し、 構造化されたリサーチレポートを作成する。 ## トリガー条件 - 「調べて：{トピック}」 - 「リサーチ：{トピック}」 - 「{トピック} についてまとめて」 ## 実行手順 1. トピックを分解して3〜5の調査観点を特定 2. 各観点について10件以上のウェブ検索を実行 3. 以下の品質基準でソースを評価・選別: - 最新性（直近1年以内を優先） - 信頼性（公式サイト・主要メディアを優先） - 多様性（視点が偏らないよう複数ソースから） 4. 情報を統合してレポートを生成 5. ~/research/{日付}_{トピック}.md に保存 6. Telegramにレポートの要約版を送信（全文はファイル参照） ## レポート構成 # {トピック} リサーチレポート 調査日: {日付} | 調査ソース数: {N}件 ## エグゼクティブサマリー（3〜5行） ## 1. 概要・定義 ## 2. 現状・最新動向 ## 3. 主要プレイヤー・事例 ## 4. 課題・リスク ## 5. 今後の展望 ## 参考ソース - [{タイトル}]({URL}) — {一言コメント} ## 注意 調査結果をそのまま事実として断言しない。 「〜と報告されている」「〜とされる」等の表現を使う。

9. GitHubコードレビュー補助
🔴 上級 Claude Sonnet or DeepSeek V3 ⏰ Push時・手動実行
PRのdiffを自動取得してコードレビューコメントを生成
# スキル名: コードレビュー補助 ## 概要 GitHubのPRのdiffを取得し、コードの問題点・改善案を 構造化されたレビューとしてまとめる。 ## トリガー条件 - 「レビュー：{PR URL}」 - 「{GitHub PR URL} をレビューして」 ## 実行手順 1. PR URLからリポジトリ名・PR番号を抽出 2. GitHub CLI または API でdiffを取得 3. 以下の観点でコードを分析: - バグ・セキュリティ上の問題 - パフォーマンス上の問題 - 可読性・命名規則 - テストカバレッジ - ドキュメント不足 4. 問題を重大度別（Critical/Warning/Suggestion）に分類 5. 各問題に具体的な改善案を付与 ## 出力フォーマット ## 🔍 コードレビュー: {PR名} **総評**: {2〜3行のサマリー} 重大度: 🔴{N}件 / 🟡{N}件 / 💡{N}件 --- ### 🔴 Critical（必須修正） **{ファイル名}:{行番号}** 問題: {問題の説明} 修正案: ```{言語} {修正後コードの例} ``` ### 🟡 Warning（修正推奨） ... ### 💡 Suggestion（改善提案） ... ## 注意 - 最終判断は人間が行う。あくまで補助ツール - 機密コードの取り扱いに注意

# スキル名: 多言語翻訳・要約 ## 概要 URLまたはテキストを受け取り、自動で言語を検出して 日本語の要約と翻訳を生成する。 ## トリガー条件 - URLが含まれるメッセージ（自動検出） - 「翻訳：{テキスト}」 - 「要約：{URL}」 ## 実行手順 1. 11. ダウンロードフォルダ自動整理
🟡 中級 GPT-4o mini ⏰ 毎週日曜22:00
散らかったDownloadsフォルダを自動分類・整理
# スキル名: ダウンロードフォルダ整理 ## 概要 ~/Downloads フォルダを定期スキャンし、 ファイルを種類・日付ごとに自動分類して整理する。 ## スケジュール 毎週日曜日 22:00 に自動実行 （「整理して」「downloads整理」で即時実行も可） ## 分類ルール - 📄 PDF → ~/Documents/PDFs/{YYYY-MM}/ - 🖼️ 画像（jpg/png/gif等）→ ~/Pictures/{YYYY-MM}/ - 📊 表計算（xlsx/csv）→ ~/Documents/Spreadsheets/ - 📝 ドキュメント（docx/pptx）→ ~/Documents/Docs/{YYYY-MM}/ - 💻 インストーラー（dmg/exe/pkg）→ ~/Downloads/Installers/ - 📦 アーカイブ（zip/tar.gz）→ ~/Downloads/Archives/ - 30日以上未アクセスの不明ファイル → ~/Downloads/Old/ ## 実行手順 1. ~/Downloads の全ファイルをスキャン 2. 拡張子と作成日でファイルを分類 3. 実行前に移動計画をユーザーに提示（承認制） 4. 承認後にファイルを移動 5. 整理サマリーをTelegramに報告 ## 重要ルール - ファイルの削除は行わない（移動のみ） - 移動前に必ず計画をユーザーに確認する - 元のファイル名は変更しない

12. タスク管理マスター（Skill連携版）
🔴 上級 Claude Sonnet ⏰ 常時稼働
タスクの記録・優先度付け・リマインダーを一元管理する中核Skill
# スキル名: タスク管理マスター ## 概要 タスクの追加・更新・完了・リマインダーを一元管理する。 他のSkillからも呼び出し可能な中核タスクマネージャー。 ## トリガー条件 - 「タスク追加：{内容}」→ タスク登録 - 「タスク一覧」「今日やること」→ 一覧表示 - 「{タスク名}完了」→ 完了マーク - 「リマインド：{日時} に {内容}」→ リマインダー設定 - 「優先度変更：{タスク名} を {高/中/低}」→ 優先度更新 ## データ保存先 ~/tasks/tasks.json （JSONで構造化保存） ## タスクデータ構造 { "id": "uuid", "title": "タスク名", "priority": "high/medium/low", "due": "YYYY-MM-DD HH:mm", "status": "todo/in_progress/done", "tags": ["仕事", "学習"], "created": "ISO8601", "reminder": "YYYY-MM-DD HH:mm" } ## 一覧表示フォーマット 📋 **タスク一覧**（{N}件） 🔴 **高優先度** - [ ] {タスク名}（期限: {日付}） 🟡 **中優先度** - [ ] {タスク名} 🟢 **低優先度 / 期限なし** - [ ] {タスク名} ✅ **本日完了** ({N}件) ## 他Skillからの呼び出し例 週次レポートSkillは本Skillのデータを参照して 完了タスク数・未完了タスクを自動集計する。
🚀 レシピをより使いこなすための3原則
  • まず1つだけ動かす： 全部入れようとせず、自分が毎日使う1つのSkillだけ試す。1週間使えたら次を追加する。
  • DeepSeekで試してClaudeで仕上げる： 初期テストはDeepSeek（低コスト）で動作確認し、品質が出るようになったらClaudeに切り替える。
  • SKILL.mdをブログ感覚で育てる： 動かしながら「ここが不満」と感じたらSKILL.mdをその場で修正。積み重ねが精度を上げる。
