Docker Essentials

The Universal Shipping Container for Code

What is Docker?

Dockerは、あなたの作ったプログラムを「どこでも動くケース」に閉じ込める技術です。
AIエンジニアにとって、これは単なるツールではなく「自律型エージェントの檻（Cage）」として機能します。

The Shipping Container Analogy

昔 (Before Docker):
引っ越しのとき、家具や食器をそのままトラックに放り込んでいました。
「Aさんのトラックでは運べたけど、Bさんのトラックだと不安定で壊れる」という問題が起きました。
(＝ PCが変わると動かない "It works on my machine" 問題)

今 (With Docker):
家具も食器も空気も、全て規格化されたコンテナに詰め込みます。
トラックの運転手（Windows, Mac, Linux）は、中身を気にせず「コンテナ」を運ぶだけ。
開ければどこでも全く同じ部屋が再現されます。

Core Concepts

🖼️ Image (設計図)

そのコンテナに何が入っているかの定義ファイル。
「Ubuntuが入っていて、Python 3.9が入っていて、このコードが入っている」という不変のデータ。 CD-ROMのようなもの。

📦 Container (実体)

Imageから生成された、実際に動いているプロセス。
1つのImageから、何個でもContainerを作れる。
使い終わったら捨てれば、跡形もなく消える（Disposable）。

Ideally for OpenClaw

なぜ自律型エージェント(OpenClaw)にDockerが必要なのか？
理由は「安全性(Safety)」と「再現性(Reproducibility)」です。

  • Sandbox (砂場): エージェントが暴走して `rm -rf /` (全削除) を実行しても、消えるのはコンテナの中だけ。あなたのPCは無傷です。
  • Consistency: 「私のPCでは動いたのに、本番サーバーではエラー」という事故を、Dockerなら100%防げます。

Basic Commands

これだけ覚えておけば十分です。

# 1. コンテナを起動する (run) # -it: インタラクティブモード # --rm: 終わったら自動削除（ゴミを残さない） $ docker run -it --rm python:3.9 # 2. 動いているコンテナを見る $ docker ps # 3. 全てのコンテナ（停止中含む）を見る $ docker ps -a # 4. コンテナを強制停止する $ docker stop
