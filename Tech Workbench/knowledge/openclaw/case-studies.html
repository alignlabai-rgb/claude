The Legacy Fortress Update
Target: Java Monolith Saved: 40 hours Cost: $18.50
The Challenge
10年前に構築された5万行のJavaシステム。ドキュメントは紛失し、当時の開発者は全員退職済み。
クライアントの要望は「この巨大な泥団子の中から、税金計算ロジック（TaxCalculator）だけを抜き出して、独立したAPIサーバーにしたい」。
人間がやれば、「コードを読む」だけで2週間はかかる絶望的なタスク。
Agent Configuration
# MISSION.md Objective: Extract `TaxCalculator` class and its dependencies to a new Spring Boot project. Constraints: - **Read-Only Mode First**: 最初はファイルを書き込まず、依存関係のリストを作ることに専念せよ。 - **No Refactoring**: ロジックを変えるとバグる可能性がある。変数名すら変えるな。コピーせよ。 - **Generate Tests**: 抽出したコードに対し、元の挙動を保証する単体テストを新規作成せよ。 Definition of Done: - 新しいプロジェクトでビルドが通る (`mvn clean install`). - テストカバレッジが 90% を超える。
Execution Process
1. Scanning: エージェントは `grep` を駆使して `TaxCalculator` から呼ばれているクラスを再帰的に特定。計12個の関連クラスを発見した。
2. Extraction: 新しいディレクトリを作成し、ファイルをコピー。しかし `UserContext` という巨大なクラスへの依存が見つかり、コンパイルエラー発生。
3. Mocking: エージェントは「全てのUserContextを持ってくるのは危険」と判断し、必要なメソッドだけを持つ Mockクラス を生成して依存を断ち切った（賢い判断！）。
🏆 Result
人間は1行もコードを読まずに、完全に独立して動作するマイクロサービスが手に入った。
生成されたテストコードのおかげで、今後の改修も安全に行えるようになった。
The Resilient Market Watcher
Target: 15 EC Sites Freq: Daily Cost: $0.80/day
The Challenge
ECサイト運営者の悩み。「競合他社15サイトの価格を毎日チェックしたい」。
従来のPythonスクレイピング（BeautifulSoup）では、相手サイトのクラス名が変わるたびにエラーになり、エンジニアが毎週修正していた。
「メンテナンスフリーな監視システム」が必要だった。
Agent Configuration
# MISSION.md Objective: Visit the URL list, find the price of "iPhone 16", and append to `prices.csv`. Constraints: - **Use Vision**: HTML構造（class名やid）に依存するな。スクリーンショットを見て、「価格っぽい数字」を目視で探せ。 - **Handle Popups**: 広告ポップアップが出たら、×ボタンを押して閉じろ。 Note: - ブラウザ操作には `puppeteer` を使用せよ。
Execution Process
1. Navigation: ブラウザを起動し、サイトAにアクセス。
2. Visual Recognition: サイトAがリニューアルされており、DOM構造が激変していた。しかしエージェントは「￥128,000」というテキストの近くにある画像を認識し、これを価格と断定。
3. Popup Handling: 突然「メルマガ登録」のポップアップが出現。エージェントは画面中央を覆う要素を検知し、右上の小さな「×」をクリックして続行した。
🏆 Result
サイトのデザイン変更でスクリプトが止まることはなくなった。
エンジニアの週1回の保守作業がゼロになり、社長は毎朝のCSVレポートを見るだけになった。
The Content Curator
Target: Tech News RSS Output: Blog Draft Cost: $2.00/run
The Challenge
技術ブログの運営者。「毎日最新のテックニュースを追いかけたいが、英語記事を読む時間が足りない」。
TechCrunch, The Verge, Hacker News の新着記事を読み、重要度を判定し、日本語で要約記事を書きたい。
Agent Configuration
# MISSION.md Objective: Read top 5 articles from Hacker News, summarize them in Japanese, and create `drafts/YYYY-MM-DD.md`. Constraints: - **Tone**: "だ・である"調。エンジニア向けの硬派な文体で。 - **Relevance**: "AI", "React", "Cloud" に関係ない記事（政治など）は無視せよ。 - **Structure**: 各記事につき「3行要約」「技術的なポイント」「著者の主張」を記載すること。
Execution Process
1. Filtering: 30本の記事タイトルを取得し、キーワードで5本に絞り込み。
2. Deep Reading: 各記事のURLにアクセスし、本文全文（Readability View）を取得。
3. Synthesis: 専門用語（RAG, LLM, Vercelなど）の解説を補足しながら、指定されたフォーマットでMarkdownを執筆。
🏆 Result
運営者は朝起きると、すでに「下書き」ができている状態になった。
あとは自分の感想を一言追記して「公開」ボタンを押すだけ。更新頻度が週1回から毎日になった。
The Vue 2 to 3 Migrator
Target: 200 Components Time: Overnight Scale: Massive
The Challenge
Vue.jsのバージョンアップ案件。Vue 2 (Options API) で書かれたコンポーネントが200個あり、全て Vue 3 (Composition API / Script Setup) に書き換える必要がある。
作業自体は単純だが、量が膨大で人間がやるとミスが出る。
Agent Configuration
# MISSION.md Objective: Convert all `.vue` files in `src/components` from Options API to Script Setup. Constraints: - **One by One**: 一気にやるな。1ファイルずつ修正し、都度 `npm test` を実行せよ。 - **Style Guide**: 変数定義は `const` を使い、リアクティブな変数は `ref` を使え。 - **Commit**: 1ファイル成功するごとに `git commit` せよ。 Definition of Done: - 全ファイルの変換が完了し、テストがAll Greenであること。
🏆 Result
エージェントは一晩かけて200ファイルを書き換えた。
途中、複雑なMixinsを使っているファイルで詰まったが、そこだけ `// TODO: Manual check` コメントを残してスキップし、他のファイルの処理を続けた（この「詰まっても止まらない」のが重要）。
The QA Impersonator
Target: Staging Env Role: User A Tool: Playwright
The Challenge
リリースのたびに「ログインできるか？」「カートに入るか？」を手動で確認するのが面倒。
E2Eテストはあるがメンテナンスされておらず、すぐ落ちる。人間が実際にブラウザを操作するようなテストがしたい。
Agent Configuration
# MISSION.md Objective: Log into Staging, buy "Item-X", and confirm the "Thank you" page appears. Constraints: - **Record Video**: 操作の様子を動画ファイルとして保存せよ。 - **Error Handling**: エラー画面が出たら、そのページのHTMLソースとスクショを保存して終了せよ。
🏆 Result
デプロイ完了のフックでエージェントが起動。
ある時、「決済ボタンが画面外にはみ出して押せない」というバグを、エージェントが「ボタンが見つかりません」というエラーで検知。
これは単体テストでは絶対に見つけられないUIバグだった。
The Academic Assistant
Target: ArXiv API Input: PDF Output: Slack
The Challenge
AI研究者。「毎日何百本も出る論文（ArXiv）の中から、自分の研究（Swarm Intelligence）に関連するものだけを見つけたい」。
キーワード検索だけでは不十分で、中身を読んで「新規性があるか」判断してほしい。
Agent Configuration
# MISSION.md Objective: Fetch today's AI papers from ArXiv, read the first 2 pages of PDF, and score them (1-10) based on relevance to "Agentic Swarm". Constraints: - **Output**: JSON format. - **Criteria**: 単なるLLMの応用は低評価。マルチエージェントの協調動作に関する新手法なら高評価。
🏆 Result
研究室のSlackに、毎朝「今日読むべき論文ベスト3」が届くようになった。
要約だけでなく「なぜこれを推薦したか（この論文は○○という点で新規性がある）」という理由付きでレコメンドされるため、精読の時間が大幅に短縮された。
