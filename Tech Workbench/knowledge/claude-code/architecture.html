Agent-First Architecture

Designing Systems for the AI Mind

「2週間で動くシステムを作る」ためには、人間にとっての読みやすさよりも、AI（Opus 4.6）にとっての理解しやすさを優先したディレクトリ構造が必要です。

1. The "Context-Friendly" Directory Structure

エージェントが迷子にならないための推奨構成です。「どこに何を置くか」の判断を迷わせないために、トップレベルは極めてシンプルにします。

/ (Project Root) ├── .ai/ # エージェント専用の記憶領域 │ ├── CONTEXT.md # [重要] プロジェクトの全貌・目的・制約条件 │ ├── RULES.md # [重要] コーディング規約 (例: "TSはstrictモードで") │ └── MEMORY.md # エージェントが自分で更新する「作業ログ・決定事項」 │ ├── modules/ # マイクロサービス的な機能区分 │ ├── auth/ # 認証機能 (独立したContextを持つ) │ │ ├── api/ │ │ ├── ui/ │ │ └── services/ │ ├── billing/ # 課金機能 │ └── core/ # 基盤ロジック │ ├── shared/ # 共通コンポーネント │ ├── ui-kit/ │ └── utils/ │ ├── tests/ # E2Eテスト └── README.md
Key Strategy: "Colocation" (関連性の凝集)
従来のMVC（Model/View/Controllerフォルダに分ける）はAIにとって最悪です。機能 A を修正するために3つの離れたフォルダを行き来させる必要があるからです。
AIには「機能単位（modules/xxx）」ですべての関連ファイルが1箇所にまとまっている構造が最強です。

2. Format of CONTEXT.md

エージェントへの「最初の神の啓示」となるファイルのテンプレートです。

# Project Phoenix Context ## 1. Goal このプロジェクトは「SaaSに依存しない自社製Core Work System」を構築することである。 最終的には社内ユーザー50名が使用する日報・承認システムとなる。 ## 2. Tech Stack - Frontend: React 19 (No Next.js), TailwindCSS - Backend: Supabase (Edge Functions) - Language: TypeScript (Strict) ## 3. Core Principles 1. **Agent-First**: コードは常にAIによる修正を前提とする。複雑な抽象化よりも、冗長でも分かりやすいコードを好む。 2. **Speed over Beauty**: UIは `shadcn/ui` をそのまま使うこと。デザインに時間をかけない。 3. **No Hidden Logic**: ビジネスロジックは全て `modules/*/services` に集約すること。UIにロジックを書かない。 ## 4. Current Phase 現在は「Authモジュール」の実装中。ユーザー登録フローを完了させることが最優先。

3. Agent-Driven Development Fow

このアーキテクチャを使って開発を進める具体的なフロー。

Phase 1: Definition

まず .ai/CONTEXT.md を人間が書きます。コードは書きません。

Phase 2: Skeleton

Claudeにコマンドを投げます。
$ claude "Read .ai/CONTEXT.md and generate the directory structure and empty files for the Auth module."

Phase 3: Logic & Test

中身を埋めさせます。
$ claude "Implement the Auth service based on best practices. Create tests first."
