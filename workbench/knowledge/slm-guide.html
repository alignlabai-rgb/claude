主要 SLM ラインナップ（2026年4月）

🌿
Bonsai 8B
PrismML（Caltech発, 2026年3月）

世界初の商用1-bit LLM。メモリ1.15GBでiPhoneに収まる超省メモリモデル。長時間エージェントタスクでFP16を逆転する。

1.15 GB 44 tok/s (iPhone) 368 tok/s (RTX 4090) Apache 2.0
超省メモリ 1-bit量子化 エッジ特化

◆
Gemma 4 E2B / E4B
Google DeepMind（2026年4月）

「Effective」パラメーター設計。スマートフォン・Raspberry Pi向けに最適化。ネイティブマルチモーダル＋ファンクションコール対応。

E2B: 1.5GB E4B: 2〜3GB 128k ctx Apache 2.0
マルチモーダル ネイティブ関数呼び出し Ollama対応
Φ
Phi-4 Mini
Microsoft（2025年〜2026年）

3.8B ながら数学・推論・コーディングで圧倒的なパフォーマンス。SambaYアーキテクチャの flash-reasoning 版は10×スループット向上。

3.8B params 128k ctx ~2GB RAM MIT
推論特化 数学・コード強い NPU最適化
🦙
Llama 3.2 (1B / 3B)
Meta（2024年後半〜）

エッジ向けの超軽量モデル。1B は Raspberry Pi Zero でも動く。エコシステムが最大で、Ollama/llama.cpp での運用実績が豊富。

1B/3B params 128k ctx 0.6〜2GB LLAMA
超軽量 エコシステム最大 実績豊富
🦙
Llama 3.1 8B
Meta（2024年〜）

汎用性と性能のバランスが最も良い中量級モデル。ローカル開発の標準的な選択肢。ファンクションコール・ツール使用に対応。

8B params 128k ctx ~5GB (Q4) LLAMA
汎用バランス ツール使用対応 FIM補完

Ollama でのセットアップ — PicoClaw と繋ぐ

Step 1: Ollama インストール

# Linux / Mac
curl -fsSL https://ollama.ai/install.sh | sh

# Windows (PowerShell)
winget install Ollama.Ollama

Step 2: モデルをプル

# Gemma 4 E2B（推奨）
ollama pull gemma4:e2b

# Bonsai 8B
ollama pull bonsai:8b-1bit-q8

# Llama 3.1 8B
ollama pull llama3.1:8b

# Phi-4 Mini
ollama pull phi4-mini

Step 3: PicoClaw の config を切り替え

{
  "provider": "openai-compatible",
  "base_url": "http://localhost:11434/v1",
  "api_key": "ollama",
  "model": "gemma4:e2b",
  "max_tokens": 4096,
  "temperature": 0.7,
  "max_tool_iterations": 15
}

Step 4: 動作確認

# Ollama が起動していることを確認
ollama serve &

# PicoClaw でテスト
picoclaw agent -m "ローカルAIでの動作テスト"

💡 コスト試算： Claude Sonnet APIを月1000回呼ぶと約$1.5〜3。同じタスクをOllama on Raspberry Pi 5で行えば電気代のみ（月数十円程度）。
