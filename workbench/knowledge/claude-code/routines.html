⏰ Routines · 2026年4月14日 Research Preview

Claude Code Routines

設定すれば、誰も呼ばなくてもAIが仕事を始める。スケジュール・API・GitHubの3トリガーでクラウド上で自律実行。Macを閉じていても止まらない。

リリース: 2026年4月14日
ステータス: Research Preview
対応プラン: Pro / Max / Team / Enterprise
実行環境: Anthropicクラウド（ローカル不要）
基本概念
Routinesとは何か — チャット型AIとの決定的な違い
「呼んだら動く」から「設定したら勝手に動き続ける」への移行。
❌ チャット型（同期モデル）
  • 画面の前に人間がいる必要がある
  • 「今日のニュースをまとめて」と毎朝手動で入力
  • 人間がオフラインになるとAIも止まる
  • 同じ作業を毎回ゼロから指示する必要がある
  • 繰り返し作業の自動化ができない
✅ Routines（非同期モデル）
  • 一度設定すれば人間不在でも実行される
  • 毎朝7時に自動でニュースまとめが完成している
  • Macを閉じていてもAnthropicクラウドで実行継続
  • プロンプト・リポジトリ・コネクターを一度設定するだけ
  • 3種のトリガーで様々な自動化が可能

Routineの定義： 保存されたClaude Codeの設定（プロンプト＋リポジトリ＋コネクター）に、1つ以上のトリガーを付けたもの。トリガーが発火すると、Anthropicのクラウドインフラ上でエージェントが自律的に実行される。

トリガーの種類
3種類のトリガー
1つのRoutineに複数のトリガーを組み合わせることもできる。
⏰ Trigger 01 — Scheduled

時間スケジュール

cron形式で繰り返し実行。「毎朝」「毎夜」「毎週」など定期的なタスクに最適。

  • 毎日・毎週・毎月・任意間隔に対応
  • タイムゾーン指定可能
  • 複数スケジュールを1つのRoutineに設定可
  • 例：0 7 * * 1（毎週月曜7時）
🔌 Trigger 02 — API

HTTP APIトリガー

各RoutineにユニークなHTTP POSTエンドポイントが発行される。外部システムから呼び出せる。

  • Bearer tokenで認証
  • リクエストボディのテキストをエージェントに渡せる
  • 例：監視アラート発火→自動調査→修正PR作成
  • 社内システムとの連携に最適
🐙 Trigger 03 — GitHub

GitHubイベント

PR作成・マージ・リリースなどのリポジトリイベントに反応して自動実行。

  • pull_request.opened：PR作成時
  • push：pushされるたびに実行
  • release.published：リリース時
  • Anthropic社内でコードレビュー自動化に使用中
セットアップ
Routineの作成方法
UIとCLIの2通りで作成できる。

① Web UI から作成

  1. claude.ai/code/routines を開く
  2. 「New Routine」をクリック
  3. プロンプトを入力（何をするRoutineか）
  4. 対象リポジトリを選択
  5. 必要なコネクターを追加（Slack・GitHub等）
  6. トリガーを設定（Schedule / API / GitHub）
  7. 保存して有効化

② CLI から作成

# Claude Code を起動して /schedule コマンドを使用
claude-code

# Routine作成ウィザードを起動
/schedule

# または直接指定
/schedule --cron "0 7 * * 1" "毎週月曜朝7時に
前週のM&Aニュースをまとめてください"
💡 確認方法

作成済みのRoutineは claude.ai/code/routines で一覧・管理・ログ確認ができる。

実践ユースケース — MAチーム向け
今すぐ実装できる7つのRoutine
グループ海外事業統括部MAチームの業務に直結するRoutineのサンプル。
📰
週次M&Aニュースモニタリング
毎週月曜8時

毎週月曜朝8時に前週のM&Aニュースを自動収集・要約。出社したらレポートが届いている。

Routineプロンプト例
Bloomberg・Reuters・日経電子版・FT で過去7日間のM&A関連ニュースを検索してください。SBIグループおよびアジア太平洋地域の金融機関に関連する案件を優先して抽出し、以下の形式で日本語レポートを作成してください：①主要案件サマリー（3〜5件）②業界トレンド総括 ③注目企業リスト。Slackの#ma-newsチャンネルに投稿してください。
🚨
ターゲット企業ニュースアラート
毎日8時・18時

ウォッチリストに登録した企業のニュースを1日2回自動チェック。重大ニュースのみ通知。

Routineプロンプト例
ウォッチリスト（watchlist.txt）に記載の企業について、過去12時間のニュースをスキャンしてください。M&A関連・財務・経営陣変更・規制対応に関するニュースがあれば、重要度（高/中/低）を付けて日本語でまとめてください。「高」の案件のみSlackに通知してください。
📋
DD初期調査の自動起動
APIトリガー

社内システムで新案件が登録されたらHTTP POSTでRoutineを起動。企業の初期調査が自動開始。

Routineプロンプト例（リクエストボディで企業名を受け取る）
{{企業名}} について以下の初期DDを実施し、dd-reports/フォルダにレポートを保存してください：①企業概要（事業・代表・設立・グループ）②直近3年の財務概況 ③主要リスク（法的・地政学・コンプライアンス）④SBIグループとの想定シナジー。出典を明記すること。完了後、担当者（pipeline.txtに記載）にSlackで通知してください。
🔍
PRの自動コードレビュー
GitHub: PR作成時

PRが作成されるたびに自動レビュー。セキュリティ・パフォーマンス・コーディング規約をチェックしてPRにコメント。

Routineプロンプト例
このPRの変更内容をレビューしてください。以下の観点でチェックし、PRにインラインコメントを追加してください：①セキュリティ脆弱性（SQLインジェクション・XSS等）②パフォーマンス問題 ③コーディング規約（CODING_STANDARDS.mdに準拠）④テストカバレッジの充足度。最後にサマリーコメントを追加してください。
📚
ドキュメント陳腐化検出
毎週金曜18時

週次でドキュメントとコードの乖離を自動検出。更新が必要な箇所を洗い出してPRを作成。

Routineプロンプト例
今週のマージ済みPRをスキャンし、変更されたAPIやロジックに対してドキュメント（docs/フォルダ）の記述が古くなっていないか確認してください。乖離を発見したら、更新案を含むPRを自動作成してください。PRには「Auto: Doc Update」というラベルを付けてください。
📊
競合他社動向レポート
毎月第1営業日

月次で主要競合銀行のAI・デジタル戦略の動向を自動リサーチ。経営層向けサマリーを作成。

Routineプロンプト例
以下の金融機関について先月のAI・デジタル投資・M&Aに関する動向を調査してください：JPMorgan・DBS・みずほ・三菱UFJ・SMBCグループ。各行の新発表・取り組み・SBIとの差分を分析し、経営層向け5枚以内のサマリースライドの骨子を作成してください。報告書として monthly-reports/フォルダに保存してください。
⚙️
本番デプロイ後の健全性チェック
APIトリガー（CD連携）

CDパイプラインがデプロイ完了後にRoutineをトリガー。エラーログ・レスポンスタイム・異常を自動確認。

Routineプロンプト例
デプロイが完了しました（バージョン: {{version}}）。過去30分のエラーログをスキャンし、新規エラーパターン・レスポンスタイム異常・前バージョンからの回帰がないか確認してください。問題が検出された場合は即座に#deploy-alertsに通知し、根本原因の仮説とロールバック可否を報告してください。問題なければ「✅ デプロイ正常」とだけ通知してください。
現状の制限（Research Preview）
今知っておくべき制限と注意点
Research Previewのため、正式リリースまでに変更の可能性がある。
⚠️
実行時間の上限：1回のRoutine実行に上限時間が設けられている（長時間タスクは分割を推奨）
⚠️
コネクター依存：Slack・GitHubなどの連携はコネクターが必要。社内専用システムとの連携はAPIトリガー経由になる
⚠️
同時実行数：同一アカウントでの並列Routine実行数に制限がある可能性（正式発表待ち）
⚠️
デバッグが難しい：非同期実行のためエラー発生時の確認は claude.ai/code/routines のログを見る必要がある
⚠️
機密データの取り扱い：Routineはクラウド上で実行される。機密性の高いデータを処理する場合はデータの匿名化や社内構築（Enterprise）を検討
💡 Research Previewでの推奨アプローチ

まずは低リスクなRoutineから始める。例：「ニュースまとめ」「ドキュメント陳腐化チェック」。慣れてきたら「DD自動調査」「PR自動レビュー」に移行するのが安全。
