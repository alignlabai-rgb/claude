Claude Computer Use

ClaudeがスクリーンショットでGUIを見て、クリック・タイプ・スクロールで操作する。「APIがない」「ログインが必要」「Excelしかない」という壁を超えてDDを自動化する次世代能力。

ステータス: GA（一般公開）
環境: Dockerサンドボックス必須
モデル: claude-sonnet-4-6（推奨）
用途: GUI操作・データ収集・フォーム入力自動化
基本概念
Computer Useとは何か — AIが「目」と「手」を持つ
従来のAIは「テキスト入出力」のみ。Computer UseはClaudeに画面を見る目とGUIを操作する手を与える。
Computer Use の動作ループ
Step 1 — 見る

スクリーンショットを取得

Claudeが画面全体のスクリーンショットを撮影してコンテキストに取り込む

↓
Step 2 — 判断する

画像を解析して次の行動を決定

「ログインボタンはここにある」「入力フィールドはここだ」とマルチモーダルで分析

↓
Step 3 — 操作する

クリック・タイプ・スクロールを実行

座標指定クリック、キーボード入力、マウスドラッグなど実際の操作を実行

↓
Step 1に戻る（ループ）

操作結果を確認

再度スクリーンショットを撮り、操作が成功したか・次に何をするかを判断

3つのコアツール

👁️ screenshot_tool

画面を見る

指定サイズのスクリーンショットを撮影してBase64エンコード画像として返す。Claudeの「目」。

  • 引数: なし（現在の画面を撮影）
  • 戻り値: Base64 PNG画像
🖱️ computer_tool

マウス・キーボードで操作する

クリック・ドラッグ・タイプ・スクロールなどのGUI操作を実行。Claudeの「手」。

  • action: "left_click", "type", "scroll", "key" など
  • coordinate: [x, y] でクリック位置を指定
⌨️ bash_tool

コマンドラインを実行する

シェルコマンド・スクリプトをサンドボックス内で実行。ファイル操作・API呼び出しなど。

  • command: 実行するbashコマンド
  • サンドボックス内のみで動作（ホストに影響なし）
セットアップ
Docker サンドボックス環境の構築
Computer UseはDockerコンテナ内の仮想デスクトップで動作する。ホスト環境への影響をゼロにするための必須要件。
⚠️ 重要: サンドボックスは必須

Computer UseをDockerなしで実際のPCデスクトップに接続しないこと。Claudeが実際のファイル・ブラウザ・アプリを操作できるため、意図しないデータ操作・削除のリスクがある。必ずサンドボックス環境内で動作させること。

# Anthropic公式Computer Useデモ環境のセットアップ
git clone https://github.com/anthropics/anthropic-quickstarts.git
cd anthropic-quickstarts/computer-use-demo

# Docker imageのビルド
docker build -t computer-use-demo .

# コンテナ起動（Xvfb仮想ディスプレイ + VNC + noVNC含む）
docker run \
  -e ANTHROPIC_API_KEY=$ANTHROPIC_API_KEY \
  -v $HOME/.anthropic:/home/user/.anthropic \
  -p 5900:5900 \   # VNC接続ポート
  -p 6080:6080 \   # noVNC Webブラウザポート
  -p 8501:8501 \   # Streamlit UIポート
  computer-use-demo

# ブラウザで http://localhost:6080 にアクセスすると仮想デスクトップが見える

Python実装の基本骨格

import anthropic
import base64
from PIL import ImageGrab  # サンドボックス内

client = anthropic.Anthropic()

def take_screenshot():
    """スクリーンショットを撮ってBase64に変換"""
    img = ImageGrab.grab()
    import io
    buf = io.BytesIO()
    img.save(buf, format='PNG')
    return base64.b64encode(buf.getvalue()).decode()

def computer_use_loop(task: str):
    """Computer Use メインループ"""
    messages = [{"role": "user", "content": task}]

    while True:
        response = client.beta.messages.create(
            model="claude-sonnet-4-6",
            max_tokens=4096,
            tools=[
                {"type": "computer_20250124", "name": "computer",
                 "display_width_px": 1280, "display_height_px": 800},
                {"type": "bash_20250124", "name": "bash"}
            ],
            messages=messages,
            betas=["computer-use-2025-01-24"]
        )

        # tool_useがなければ完了
        if response.stop_reason == "end_turn":
            break

        # ツール呼び出しを処理してループ継続
        tool_results = process_tool_calls(response)
        messages.append({"role": "assistant", "content": response.content})
        messages.append({"role": "user", "content": tool_results})
MAチーム活用
DDにおけるComputer Use活用シナリオ
「APIがない」「ログインが必要」という壁を超えて自動化できる。
🏦
シナリオ1: データルームポータルからの自動資料収集
高価値

IntralinksやAnsaradaのようなデータルームポータルはAPIを提供していない。Computer Useを使えば、ブラウザでログイン→フォルダ階層をナビゲート→全PDFをダウンロードする操作を自動化できる。

  • ログイン（ID/PW入力、2FA対応）
  • ドキュメントリストの取得（スクリーンショット解析）
  • 全カテゴリのPDFを順番にダウンロード
  • ファイル名・フォルダ構造をメタデータとして記録
  • Files APIへ自動アップロードして分析エージェントへ受け渡し
📊
シナリオ2: 公開財務データの自動収集
定期実行可

EDINETやSECのEDGARから対象企業の有価証券報告書・10-K・プレスリリースを自動収集。ログイン不要の公開サイトならRoutinesと組み合わせて定期収集も可能。

  • 検索フォームに企業名・証券コードを入力
  • 検索結果から最新の報告書リンクを特定・クリック
  • PDFダウンロードボタンを押す
  • 複数企業を順番にループ処理（監視リスト全社を一括処理）
📋
シナリオ3: Excelフォームへの自動入力・出力
実用的

社内のExcelベースのDD進捗管理表や、顧問弁護士から送られてくるExcelフォームへの記入作業を自動化。APIやPythonで直接操作できないマクロ付きExcelでも対応可能。

  • ExcelファイルをDockerコンテナ内のLibreOfficeで開く
  • AI分析の結果を該当セルにタイプして入力
  • 保存・PDF出力まで一連の操作を実行
  • 完成したファイルをホスト環境へ受け渡し
🔍
シナリオ4: 競合他社・ターゲット企業の定期モニタリング
Routines連携

Bloomberg Terminal・Capital IQのようなデスクトップアプリや、スクレイピング禁止サイトの情報を、「人間の代理として閲覧・転記」という形で収集する。毎朝Routinesで起動して自動レポート生成まで。

  • アプリケーション起動・ログイン
  • 特定銘柄の最新データをスクリーンショットで取得
  • 画像内のテキストをOCRで読み取り構造化
  • 前日比変化点をフラグアップしてSlack通知
制約と注意事項
できること・できないこと
Computer Useには重要な制約がある。過度な期待を持たず、適切なユースケースを選ぶことが成功の鍵。
項目
✅ できること
❌ できないこと・難しいこと
ナビゲーション
シンプルなサイト・アプリのページ遷移
CAPTCHA・reCAPTCHAの突破
データ収集
公開Webサイトのフォームや一覧からの読み取り
JavaScriptで動的生成される無限スクロールコンテンツの完全収集
ファイル操作
ダウンロード・アップロード・ファイル名変更
暗号化PDFのロック解除（非公開情報へのアクセス）
フォーム入力
テキスト・数値・ドロップダウン選択
極めて複雑な動的フォーム（順序依存のある高度なUI）
速度
人間と同等またはそれ以下のペース
高速スクレイピング（ボット検知の対象になる）
信頼性
確認ステップを組み込めば高精度
UI変更（ボタン位置の変更等）に自動追従するのは難しい場合がある
⚠️ 法的・倫理的注意事項

利用規約でスクレイピングを禁止しているサイトでの使用は違反になる可能性がある。データルームの場合はNDA・利用許諾の範囲内での使用を確認すること。「人間が手作業でする操作をAIが代行する」という範囲に留めること。

コスト見積もり

スクリーンショット1枚 ≈ 1,000〜2,000トークン。操作が複雑なほどターン数が増えてコストが上昇する。1つのタスク（30操作程度）でSonnet換算 $0.05〜0.20 程度。重要なのは確認ステップを組み込んで無駄なループを防ぐこと。

ベストプラクティス
安全な実装のための設計パターン
確認ステップと人間監視を組み込んだ「Human-in-the-loop」設計が推奨される。
# 安全なComputer Use実装 — 確認ステップ付き
import anthropic

client = anthropic.Anthropic()

def safe_computer_use(task: str, require_confirmation: bool = True):
    """
    MAチーム向け安全設計のComputer Use実装
    - 重要操作前に確認を求める
    - ファイルダウンロードのみ許可（アップロード・送信は別確認）
    - 操作ログを全記録
    """
    system_prompt = """
    あなたはMAチームのデータ収集補助エージェントです。
    以下のルールを厳守してください：

    【許可する操作】
    - Webサイトの閲覧・スクロール
    - ファイルのダウンロード
    - フォームへの読み取り専用アクセス

    【禁止する操作】
    - メールの送信・フォームの送信（SUBMIT不可）
    - ファイルの削除・移動
    - アカウント設定の変更
    - ダウンロード以外のアップロード

    重要な操作（ダウンロード、ログイン等）を実行する前に、
    必ず「これから[操作内容]を行います。よろしいですか？」と確認すること。
    """

    messages = [{"role": "user", "content": task}]
    operation_log = []

    while True:
        response = client.beta.messages.create(
            model="claude-sonnet-4-6",
            max_tokens=4096,
            system=system_prompt,
            tools=[
                {"type": "computer_20250124", "name": "computer",
                 "display_width_px": 1280, "display_height_px": 800},
                {"type": "bash_20250124", "name": "bash"}
            ],
            messages=messages,
            betas=["computer-use-2025-01-24"]
        )

        # 操作ログに記録
        for block in response.content:
            if hasattr(block, 'type') and block.type == 'tool_use':
                operation_log.append({
                    "tool": block.name,
                    "input": block.input,
                    "timestamp": datetime.now().isoformat()
                })

        if response.stop_reason == "end_turn":
            break

        # ツール結果を処理してループ継続
        tool_results = process_tool_calls(response)
        messages.append({"role": "assistant", "content": response.content})
        messages.append({"role": "user", "content": tool_results})

    return {"result": response.content, "log": operation_log}

🔒 最小権限原則

Skillとして定義する際にtoolsを明示的に制限する。Computer Useはデータ収集専用Skillに限定し、財務分析Skillとは分離する。

📋 操作ログの記録

全操作をタイムスタンプ付きで記録する。「何をいつダウンロードしたか」の監査証跡が、コンプライアンス上も重要になる。

👁️ Human-in-the-loop

初回実行は必ず人間が監視する。noVNCでリアルタイムに画面を確認しながら動作を観察し、問題がないことを確認してから自動化する。
