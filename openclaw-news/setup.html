ステップ1：インストール

方法A：標準インストール（推奨）

1
OpenClawリポジトリをクローン
🪟 Windows 🍎 macOS 🐧 Linux
# GitHubからクローン git clone https://github.com/openclaw/openclaw.git cd openclaw # Pythonの依存関係をインストール pip install -r requirements.txt # または uvx（高速）を使う場合 uvx openclaw
2
Dockerを使う場合（環境分離・推奨）
# Docker経由で起動（依存関係を自動解決） docker run -d \ -v ~/.openclaw:/root/.openclaw \ -e ANTHROPIC_API_KEY=your_key_here \ --name openclaw \ ghcr.io/openclaw/openclaw:latest # ログを確認 docker logs openclaw -f

方法B：Nanobot（軽量版・Docker不要）

💡 Nanobotとは

OpenClawの軽量代替。Pythonコード4,000行のシンプル設計で、Raspberry Piでも動作。Docker不要でインストールが簡単。OpenClawに慣れる前の入門としても最適。

# Nanobotをインストール pip install nanobot # 起動 nanobot start --llm claude --api-key your_key_here

ステップ2：APIキーの設定

1
設定ファイルを作成

OpenClawは ~/.openclaw/config.yaml で設定を管理します。

# 設定ディレクトリを作成 mkdir -p ~/.openclaw # 設定ファイルを作成（以下をコピーして編集） cat > ~/.openclaw/config.yaml << EOF llm: provider: anthropic # anthropic / openai / deepseek model: claude-sonnet-4-6 api_key: YOUR_API_KEY_HERE # コスト上限を必ず設定！（月額ドル） limits: monthly_budget: 20 daily_budget: 5 messaging: provider: telegram # telegram / discord / signal token: YOUR_BOT_TOKEN EOF
⚠️ 重要：コスト上限を必ず設定すること

デフォルトではコスト上限がありません。自律実行が意図せず多くのAPIを消費し、一夜で$1,100（約17万円）の請求が発生した事例があります。monthly_budget と daily_budget は必ず設定してください。

ステップ3：Telegramボット接続（最も簡単）

1
BotFatherでボットを作成
  1. Telegramで @BotFather を開く
  2. /newbot を送信
  3. ボット名を入力（例：MyOpenClaw）
  4. ユーザー名を入力（例：my_openclaw_bot）
  5. 発行された APIトークン（123456789:ABC...）をコピー
2
設定ファイルにトークンを追加して起動
# config.yaml の messaging.token に貼り付け messaging: provider: telegram token: 123456789:ABCdefGHIjklMNOpqrsTUVwxyz # OpenClawを起動 openclaw start # Telegramでボットを開いて、試しに送信 → 「今日のTo-Doリストを作って」 → 「明日の天気を調べて教えて」
✅ 動作確認：最初に試すコマンド
  • 「現在の時刻と今日の日付を教えて」（基本動作確認）
  • 「Googleで『OpenClaw 最新情報』を検索して要約して」（ウェブ検索）
  • 「このメッセージを10分後にリマインドして」（スケジュール管理）
  • 「今日の作業ログを ~/notes/today.md に書いて」（ファイル操作）

ステップ4：Skillを自作する — 核心の理解

OpenClawの真の力は「Skillシステム」にあります。自分でSkillを作れると、どんなタスクでも自動化できるようになります。

Skillの基本構造

各Skillは以下の構造を持つディレクトリです：

skills/ └── my-skill/ ├── SKILL.md # 必須：スキルの説明・使い方・ツール定義 ├── main.py # 任意：Pythonスクリプト ├── config.yaml # 任意：設定ファイル └── README.md # 任意：ドキュメント

SKILL.mdの書き方

SKILL.md がSkillの「脳」です。LLMがこのファイルを読んで、いつ・どうやってSkillを使うかを判断します。

📄 SKILL.md の基本テンプレート
# スキル名: 日経ニュース要約 ## 概要 日本経済新聞のRSSフィードから最新ニュースを取得し、 重要トピックを3行で要約してSlackに送信するスキル。 ## トリガー条件 以下のメッセージを受け取った時に起動する: - 「今日のニュースを教えて」 - 「経済ニュースを要約して」 - 「日経チェック」 ## 使用ツール - web_search: ニュースの取得 - file_write: ログの保存 - message_send: 要約の送信 ## 実行手順 1. 日経電子版のRSSフィード(https://www.nikkei.com/rss/...) を取得 2. 上位5件のタイトルと要約を抽出 3. 各記事を3行以内で日本語要約 4. 「📰 本日のニュース要約」としてメッセージ送信 5. 実行ログを ~/notes/news_log.md に追記 ## 出力フォーマット 📰 **本日の主要ニュース（{日付}）** 1. **{見出し}** {3行要約} 2. **{見出し}** {3行要約} ...

実践：実際に使えるSkillを3つ作ってみよう

📧
Skill例1：メール日次レポート

毎朝9時にGmailの未読メールを確認し、重要度別に分類して要約をTelegramで送信。

# SKILL.md のトリガー設定例 ## スケジュール - 毎日 09:00 に自動実行 - 「メール確認して」で即時実行
📊
Skill例2：為替・市場モニター

USD/JPYが設定レートを超えたらアラート。週次で主要指数のサマリーを作成。

# SKILL.md のトリガー設定例 ## トリガー条件 - USD/JPY が 150円を超えた時 - 毎週金曜 18:00（週次レポート）
📝
Skill例3：学習メモ整理

「メモ：{内容}」と送るだけで自動分類・タグ付けしてObsidianノートに追記。

# SKILL.md のトリガー設定例 ## トリガー条件 - 「メモ：」で始まるメッセージ - 「ノート保存：」で始まるメッセージ

Skillをデプロイする

# Skillディレクトリをに配置する場所 # 1. ワークスペースSkill（特定プロジェクト専用、最優先） ./workspace/skills/my-skill/SKILL.md # 2. グローバルSkill（全ワークスペースで共有） ~/.openclaw/skills/my-skill/SKILL.md # 3. 再起動してSkillを読み込み openclaw restart # 4. Skill一覧を確認 openclaw skills list
🎯 Skill作成のコツ
  • SKILL.mdは具体的に書く： 「いつ実行するか」「どのデータを取得するか」「どんな形式で出力するか」を明確に。曖昧だとLLMが意図しない動作をする
  • 最初は小さく作る： 複雑なSkillより、1つのことだけをする単純なSkillから始める
  • ログを必ず取る： file_write でログを残すと、何が起きたか追跡できる
  • テストメッセージを用意する： SKILL.mdにテストコマンド例を書いておくと開発がスムーズ
  • エラーハンドリングを明示： 「失敗した場合は〇〇してください」と書いておく
