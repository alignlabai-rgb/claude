Part 1 — Skills
⚡ Skills — チームの「専門能力」を制度化する
Skillsは単なる便利なプロンプト保存ではない。チームの標準能力をシステム化して、誰が使っても・何度使っても同じ品質で出力させる仕組みだ。
❌ Skillsなし（今の状態）
  • 「DDメモを作って」→ 毎回フォーマットがバラバラ
  • 「社内の稟議書の書き方で」と毎回説明が必要
  • 高橋さんのプロンプトが優秀だが、チームと共有できていない
  • 新人が使うとベテランと出力の質が大きく違う
  • プロンプトは個人のメモ帳にしかない
✅ Skillsあり（導入後）
  • 「DDメモを作って」→ 必ず社内フォーマットで出力される
  • フォーマット・トーン・チェック項目はSkillに一度書けば永続
  • 最高のプロンプトをSkillとして保存 → チーム全員が同じ品質
  • 新人が使っても、Skillがあればベテランと同じ水準
  • Skillはチームの資産。退職者が出ても知識が残る

Skillsの正体 — 「MAチーム専用の記憶」をAIに持たせる

SkillsはMarkdownファイル（SKILL.md）で定義する。Claudeはこのファイルを読み込んで、そこに書かれたルール・知識・手順を「確実な事実」として持つ。推測や創作をしなくなる。

Skillsに書けること（何でもOK）
  • 社内ディールメモの構成・セクション名・書き方
  • SBIグループとしてのコンプライアンス確認項目
  • FCPA・贈賄防止法の適用チェックリスト
  • リスク分類の基準（高/中/低の定義）
  • 役員向け資料のトーン・禁止表現・推奨表現
  • ターゲット企業への英文メール文体ルール
  • DDチェックリストの全項目と優先順位
Skillsを使うと何が変わるか
  • 「ターゲット企業名」だけ入力すればDDメモが完成する
  • フォーマットの確認・修正作業がゼロになる
  • コンプライアンスチェック漏れがなくなる
  • 日英バイリンガル版も同時に生成できる
  • 5人のチームが5通りの書き方をする問題が消える

Skillsの設定方法 — コードは一切不要

Claude.aiの「Project」機能を使えば、今すぐSkillsを設定できる。開発者は必要ない。

今すぐできる設定方法（claude.ai）
  • Project を作成：「MAチーム DDワークフロー」と名前をつける
  • System Prompt にルールを記入：社内フォーマット・トーン・チェック項目を貼り付ける
  • チームメンバーを招待：全員が同じProjectを共有
  • 以降は全員がそのProjectで会話：Skillが自動で適用される
所要時間

初期設定は30分〜1時間。一度設定すれば、チームメンバーが増えても設定不要。

Claude Code を使った高度な設定（将来）
  • .claude/agents/ フォルダに SKILL.md を配置
  • 複数のエージェントに異なるSkillを割り当てられる
  • DDエージェントにはDDのSkill、法務エージェントには法務のSkill、と役割分担できる
  • Sub-agentsとSkillsを組み合わせることで精度が大幅に向上
重要な原則

「良いSkillを持ったエージェント」は、「高品質なプロンプトを毎回書ける人」と同等の能力を持つ。Skillを整備することは、チームの底上げに直結する。

Part 2 — Sub-agents & Agent Teams
🤖 サブエージェント — 1人が無限のチームを指揮する
「1対1のAI会話」から「1人が∞のAIチームを並列指揮する」への根本的な変化。MAチームのDDワークフローが、人員を増やさずに10倍速くなる。

「サブエージェント」と「Agent Teams」— 2つの使い分け

⬡ サブエージェント（Sub-agents）

メインのClaudeが子エージェントを生み出して指示する。子同士は直接会話しない。独立したタスクを並列で実行するのに最適。

  • 「同じ調査を10社に並列でかける」
  • 「企業概要・財務・リスクを同時に別々で調べる」
  • 各エージェントの結果が独立していてOKな場合
並列・独立タスク

✦ Agent Teams（チーム型）

複数のエージェントが独立したセッションで動き、互いに結果を受け渡す。DDのように「前のエージェントの結果を次のエージェントが使う」場合に最適。

  • 「各担当の調査結果を最後に統合してメモを作る」
  • 「Writerが書いたものをReviewerがチェックして修正する」
  • 前工程の成果物を次工程が必要とする場合
連携・統合タスク

Part 3 — Skills × Sub-agents の組み合わせ
🔗 2つを組み合わせると何が起きるか
SkillsとSub-agentsは単体でも強力だが、組み合わせることで「専門知識を持ったエージェントチームが並列で動く」という最強の状態になる。

組み合わせの本質

Skills単体の状態
  • 1人のエージェントが社内フォーマットを知っている
  • でも調査は1件ずつ、順番に実行される
  • 完了まで時間がかかる
Sub-agents単体の状態
  • 複数が同時に動いて速い
  • でも各エージェントは「一般的なAI」として動く
  • 出力のフォーマットや品質にバラつきが出る
✦ 組み合わせたときの状態

「DDの専門知識とフォーマットを持ったエージェントが5体、同時に並列で動いて5分でディールメモを完成させる」。速くて、正確で、フォーマット通り。これが最終形。

組み合わせ例 1

DDメモ生成Skill × DDスウォーム
  • Agent A〜Dそれぞれに「DDメモ生成Skill」を持たせる
  • 各エージェントが専門分野でSkillのフォーマット通りに調査
  • Agent Eが統合するとき「繋ぎ目がない」完璧なメモができる

組み合わせ例 2

コンプライアンスSkill × レビューLoop
  • Writerがドキュメントを作成
  • ReviewerエージェントがコンプライアンスSkillを持っていて法的チェックを実行
  • FCPAチェック漏れゼロが保証された状態でドキュメントが完成

組み合わせ例 3

モニタリングSkill × Routines
  • モニタリングSkillを持ったエージェントが毎週月曜に自動起動
  • 決まったフォーマットでM&Aニュースレポートを生成
  • 「誰も頼んでいないのに、毎週完成している」状態

💡 プロンプトをもっと良くする1つのコツ

「何を出力してほしいか」だけでなく、「なぜその情報が必要か」をプロンプトに加えると精度が大幅に上がる。例：「リスク抽出してください」より「SBIグループとしての買収判断に使うため、リスクを高/中/低で分類してください」の方が、判断に使える出力になる。

FAQ
よくある疑問
サブエージェントとSkillsについての典型的な質問と回答。

Q: サブエージェントを使うには特別なスキルが必要ですか？

claude.aiのタブを複数開いて、それぞれに別の指示を入れる方法であれば今すぐ・誰でも・コード不要で始められます。より高度な自動化（自動的にサブエージェントを生成・管理）はClaude Codeという開発ツールが必要ですが、デモで見せる「複数タブで並列実行」は完全にノーコードです。

Q: Skillsを作るには開発者が必要ですか？

claude.aiのProject機能のSystem Promptを使う方法なら開発者不要です。社内フォーマット・ルール・チェックリストをテキストで貼り付けるだけ。より高度な設定（Markdownファイルで複数エージェントに異なるSkillを割り当てる）はClaude Codeが必要ですが、まずはProjectのSystem Promptから始めて、効果を確認してから高度化するのが現実的な進め方です。

Q: 出力の精度は信頼できますか？

Skillsで正確な情報・フォーマット・確認項目を定義することで、一般的なAIよりも大幅に精度が向上します。ただし「初期ドラフト・調査のたたき台」として使い、人間が最終確認するワークフローが現実的です。特に財務数値・法的判断は出典確認と専門家レビューを組み合わせることを推奨します。サブエージェントによる並列調査で調査カバレッジは上がるため、確認漏れは減ります。

Q: 機密情報を入れても安全ですか？

Claude.aiのTeam/Enterpriseプランでは会話データがAIの学習に使われません。ただし入力データはAnthropicのサーバーを通ります。現実的な運用は「社名・個人名を伏せて匿名化した形で初期調査に使い、具体的な機密情報の取り扱いは社内ポリシーに従う」というアプローチです。将来的にはAPIを使った完全社内環境構築も可能です。
