GitHub Copilot

開発ライフサイクルのすべてに偏在（Ubiquitous）するパートナー。
エディタ内の補完から、PRレビュー、Issue整理、そして「Copilot Workspace」による自律コーディングまで、GitHubエコシステム全体をAI化します。

1. Copilot Workspace CORE

「IssueからPRまで」を自律的にこなすエージェント機能です。

📝 Issue理解: ユーザーがIssueを立てると、その意図と関連コードを自動解析。
💡 Plan策定: 「どのファイルのどこを直すか」という計画を自然言語で提案。
✨ Implement: エディタを開かずに、ブラウザ上またはVS Code上でコードを自動生成し、修正案を提示。

Claude Codeが「対話型」なら、Workspaceは事前にタスク構造化を行う「仕様書駆動型」のアプローチです。

2. Copilot in VS Code

最も馴染み深い、エディタ統合型の機能です。

  • Ghost Text: カーソルの動きを読んで、数行先のコードをグレー文字で先読み提案（Completion）。
  • Inline Chat: Cmd+I でエディタ内にチャット欄を呼び出し、「この関数をリファクタリングして」と指示。
  • Next Edit Suggestions: 「ここを直したなら、あっちも直すべき」という関連個所へのカーソルジャンプ（Copilot Edits）。

3. Comparison: Copilot vs Claude Code

どちらも「エージェント」ですが、得意な距離感が異なります。

🚀 GitHub Copilot

  • エコシステム重視: GitHub IssueやPRとの連携が最強。
  • 補完タイプ: ユーザーが書いている最中の「先読み」が得意。
  • Workspace: タスク単位でまとめて修正案を作る。

🧠 Claude Code

  • 思考力重視: 複雑な設計や、答えのない問題への壁打ちが得意。
  • 対話タイプ: ユーザーと会話しながら深掘りする。
  • CLI/MCP: ローカル環境や外部ツールを自由に組み合わせて使う。

Vibe Codingでの使い分け

「Copilotは手足（と反射神経）、Claude Codeは参謀」

  • 日常的なコーディング、定型的なバグ修正、PR作成 → GitHub Copilot に任せる。
  • 新しいアーキテクチャの検討、原因不明の難解なバグ、仕様策定 → Claude Code と相談する。

両者は競合するものではなく、Level 2のエージェントレイヤーにおいて補完し合う関係にあります。
