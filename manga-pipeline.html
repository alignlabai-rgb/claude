ビジュアル制作エンジン

漫画パイプラインは動画制作と並列に稼働する第2の収益エンジンです。 同じネタ文章から漫画・電子書籍を同時生成することで、コンテンツの多角的収益化を実現します。 Kindle・note・Boothへの出力を担う最終ビジュアル製造ラインです。

  • 入力元: ネタ生成LLM（ストーリーテキスト）
  • 出力先: 評価システム（PDF/EPUB/CBZ + メタデータ）
  • 処理時間: 8ページ漫画 約20〜40分
  • 必要GPU: VRAM 12GB以上（LoRA + ControlNet同時使用）
  • キャラ管理: 最大10キャラのLoRAを並列管理

モジュール入出力定義

# OpenClaw スキル定義 (SKILL.md 抜粋) name: manga-pipeline version: "1.8.0" input: story_text: string # ストーリー原稿 genre_preset: string # shonen|romcom|mystery page_count: int # 目標ページ数 characters: list # キャラ設定リスト color_mode: string # mono|color output: pdf_path: string # 漫画PDFパス epub_path: string # Kindle用EPUB preview_images: list # サムネ用画像 metadata: object # タイトル/説明/タグ cost_usd: float
最重要課題

キャラクターの
一貫性をどう保つか

AI漫画制作で最大の課題は「同じキャラクターを複数コマにわたって一貫して描くこと」。 毎回異なる顔になってしまうと漫画として成立しません。

  • LoRAモデルでキャラを事前学習
  • シード値・プロンプトの徹底管理
  • IP-Adapter でリファレンス画像を維持
  • キャラ設定シートをプロンプトに常時注入
  • 差分のみ生成して顔パーツを固定

制作フロー

6ステップで完成漫画へ

1

ストーリー構造化

ネタ文章をシーン・セリフ・感情・カメラアングルに構造化。LLMが漫画的な「間」や「演出」を考慮した構成に変換。

// 構造化出力例 { "scene": 3, "panels": [ { "angle": "close-up", "emotion": "surprise", "dialogue": "えっ、本当に？", "sfx": "ドキッ" } ] }
2

コマ割り設計

ページあたりのコマ数・サイズ・配置をAIが自動設計。テンション高い場面は大ゴマ、説明場面は小ゴマを自動判断。見開きや見せ場コマも自動挿入。

Claude Sonnet 4.6 / Gemini 2.x (構成判断) カスタムレイアウトエンジン
3

キャラクター・背景生成

LoRA + IP-Adapterで一貫したキャラを生成。背景はControlNet + 深度マップで遠近感を保持。白黒漫画風・カラー漫画風の切り替えも対応。

Stable Diffusion XL ControlNet IP-Adapter ComfyUI
4

吹き出し・セリフ配置

セリフの長さ・感情に合わせた吹き出し形状を自動選択（通常・叫び・思考・ナレーション）。フォント・サイズ・配置をコマ内の空白スペースに最適化。

Pillow (Python) カスタム吹き出しSVG 自動レイアウト調整
5

効果線・トーン追加

スピード線・集中線・背景トーン（スクリーントーン）を自動追加。シーンの感情に合わせた効果を自動判断して適用。

カスタムComfyUIノード OpenCV (線処理)
6

ページ結合・出力

全コマをページにレイアウトし、PDF・CBZ・Web閲覧用HTMLとして出力。電子書籍配信用のメタデータも自動生成。評価システムへ送付。

PDF出力 Kindle MOBI/EPUB Web漫画HTML
LoRA準備フロー & 一貫性達成率

キャラクターLoRAを
パイプラインに組み込む

LoRAは漫画生成前に1回だけ学習・登録します。 登録済みLoRAはOpenClawが管理し、ジャンル・キャラクターに応じて自動選択します。 学習データの品質が一貫性に直結するため、準備プロセスが重要です。

LoRA準備の4ステップ

1
参照画像収集
キャラクター設定画20〜50枚（正面・斜め・表情バリエーション）をMidjourneyで生成
2
LoRA学習（Kohya SS）
SDXL上でステップ数2000〜3000、学習率1e-4。RunPodのA100で約20〜40分・費用約¥300〜600
3
品質テスト & シード確定
10シードでテスト生成。最も安定するシードをキャラ設定に固定登録
4
OpenClawへ登録
SKILL.md のキャラクターリストに追加。以降は自動でロードされ人間の操作不要

技術別 一貫性達成率（設計目標値）

LoRA単体65〜75%
LoRA + シード固定80〜87%
LoRA + IP-Adapter88〜93%
LoRA + IP-Adapter + ControlNet Canny93〜97%

※ 一貫性 = 同一キャラクターを人間が見て「同一人物」と判定できる割合。測定はCLIP特徴量コサイン類似度0.85以上を基準として算出（設計目標値）。

複数キャラ同フレームの対策
キャラAのLoRA重み0.7 + キャラBのLoRA重み0.5で合成。衣装・髪色の差が大きいほど成功率が向上。類似キャラの同時登場は避けるか、手動レビュー対象とする。

連携設計

動画パイプラインとの
並列実行アーキテクチャ

同じネタから動画・漫画を並列生成することで制作効率を最大化します。 共通の「ストーリー構造化」結果を両パイプラインで共有し、 APIコストと処理時間を削減します。

# 並列生成ワークフロー (OpenClaw YAML) workflow: dual-content-creation steps: - name: structure-story # 共通前処理 skill: story-analyzer - name: create-video skill: video-pipeline parallel: true input: story: "${{structure-story.output}}" - name: create-manga # 同時実行 skill: manga-pipeline parallel: true input: story: "${{structure-story.output}}" - name: evaluate-both skill: content-evaluator depends_on: [create-video, create-manga]

技術スタック詳細

ComfyUIワークフロー構成

ComfyUIをバックエンドとして使用し、各生成ノードをAPIで制御。キャラクター別にLoRAを管理し、シーンごとに自動切替。

  • ComfyUI APIでバッチ生成を自動化
  • キャラ×感情×角度のプロンプトマトリクス管理
  • ControlNet Cannyで線画の一貫性を確保
  • ADetailerで顔の品質を自動修正
  • 生成失敗時の自動リトライ機能
