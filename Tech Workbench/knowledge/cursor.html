なぜ Cursor なのか？

VS Codeの拡張機能としてAIを入れるのではなく、エディタ自体がAIを前提に作られています。

  • VS Code完全互換: 設定や拡張機能をそのまま引き継げます。
  • Codebase Indexing: プロジェクト全体を理解し、的確な提案を行います。
  • Composer: 複数ファイルを同時に編集・生成する強力な機能。

必須ショートカット

  • Cmd + K : エディタ内でAIに指示（「この関数をリファクタリングして」）。
  • Cmd + L : チャットパネルを開く（「このコードの意味は？」）。
  • Cmd + I : Composer (Beta) を起動。複数ファイルにまたがる変更を一括指示。
  • Tab : AI提案の確定（Copilotより強力なオートコンプリート）。

Vibe Coding との関係

「1行ずつ書く」時代は終わりました。Cursorを使えば、あなたは「何を作りたいか」を自然言語で記述し、Tab キーでそれを承認する指揮者になります。

特に Composer 機能は、Vibe Codingの「全体感を伝えて細部を任せる」スタイルと完全に合致します。
