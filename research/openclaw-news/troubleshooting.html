よくある問題とその解決方法

1. インストール・起動の問題

「openclaw: command not found」と表示される

原因： PATHにOpenClawのインストールディレクトリが含まれていない

解決策： 
# PATHを確認
echo $PATH

# OpenClawのパスを追加（bash）
echo 'export PATH="$HOME/.openclaw/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc

# zshの場合
echo 'export PATH="$HOME/.openclaw/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc
「Port 3457 already in use」エラー

原因： 別のOpenClawインスタンスまたは他のプロセスがポートを使用中

解決策： 
# 使用中のプロセスを確認
lsof -i :3457

# 別のポートで起動
openclaw start --port 3458

# 既存プロセスを停止
openclaw stop
Node.jsバージョンエラー

原因： OpenClawはNode.js 18以上を要求しますが、古いバージョンがインストールされている

解決策： 
# 現在のバージョンを確認
node --version

# nvmでNode.js 20をインストール
nvm install 20
nvm use 20

2. API接続の問題

「API key invalid」エラー

原因： APIキーが正しく設定されていない、または期限切れ

解決策： 
# APIキーの設定を確認
openclaw config show

# APIキーを再設定
openclaw config set --provider claude --api-key YOUR_KEY

# 接続テスト
openclaw test-connection
「Rate limit exceeded」エラー

原因： LLMプロバイダーのレート制限に到達

解決策：
  • タスクの同時実行数を減らす：openclaw config set --max-concurrent 3
  • リクエスト間の遅延を増やす：openclaw config set --request-delay 1000
  • バックオフ設定を調整：openclaw config set --retry-backoff exponential

3. Skillの問題

Skillが認識されない

原因： SKILL.mdの形式エラー、またはSkillディレクトリのパスが正しくない

解決策： 
# Skillの検証
openclaw skill validate ./my-skill

# Skillのリロード
openclaw skill reload

# インストール済みSkill一覧を確認
openclaw skill list
Skill実行時にタイムアウト

原因： 外部サービスへの接続遅延、またはタスクが複雑すぎる

解決策：
  • タイムアウト値を増やす：openclaw config set --skill-timeout 120
  • 複雑なタスクは分割して実行する
  • 外部サービスの接続状況を確認する

4. メッセージング連携の問題

Signalボットが応答しない

原因： signal-cliのセットアップ不備、またはSignalアカウントの認証切れ

解決策： 
# signal-cliのステータス確認
signal-cli -u +YOUR_NUMBER receive

# 再認証
signal-cli -u +YOUR_NUMBER register
signal-cli -u +YOUR_NUMBER verify CODE

# OpenClawのSignal設定を確認
openclaw config show --section messaging

5. パフォーマンスの問題

🐌 OpenClawの動作が遅い場合
  • メモリ不足： openclaw statusでメモリ使用量を確認。512MB以上を推奨
  • ログ肥大化： openclaw logs rotateでログファイルをアーカイブ
  • キャッシュクリア： openclaw cache clearで古いキャッシュを削除
  • 同時タスク数： --max-concurrentを下げてシステム負荷を軽減

6. セキュリティ関連の問題

🚨 「Potential security threat detected」警告
  • 処理中のデータにプロンプトインジェクションの疑いがある場合に表示される
  • 対処：該当タスクを停止し、入力データを確認
  • セキュリティログを確認：openclaw logs --filter security
  • 不審なSkillがインストールされていないか確認：openclaw skill audit

7. ログの確認方法

# リアルタイムログ表示
openclaw logs --follow

# エラーのみ表示
openclaw logs --level error

# 特定の日付のログ
openclaw logs --date 2026-03-21

# JSON形式で出力（分析用）
openclaw logs --format json > logs.json
