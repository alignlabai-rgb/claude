LEVEL 2 : AGENT

Claude Code

単なる「賢いチャットボット」ではありません。Claude Codeは、あなたの端末内でシェル、エディタ、Git、そして外部APIを自在に操る、「もう一人の熟練エンジニア」 です。
2026年、その能力は「Sub-Agent」と「Skills (MCP)」によって飛躍的に進化しました。

Core Capabilities

1. Deep Research

「エラーの原因を調べて」の一言で、Claude Codeは動き出します。

  • Recursive Grep: 関連ファイルを自律的に探索し、影響範囲を特定。
  • LSP Integration: VS CodeのLSP（Language Server Protocol）と連携し、型定義や参照箇所を正確に把握。
  • Log Analysis: 数万行のログファイルから異常値を検出。

2. Architecture Planning

コードを書く前の「設計」こそが彼の真骨頂です。

  • Design Doc Gen: 要件を聞き出し、Markdown形式の詳細設計書を自動生成。
  • Mermaid Diagrams: クラス図やシーケンス図を描画し、視覚的なレビューを提案。
  • Refactor Strategy: 「この機能を実装するには、まずここをリファクタリングすべきです」といった戦略的提案。

3. Execution & Verification

計画を実行に移し、テストまで完遂します。

  • Auto-Testing: 修正に対応するテストコードを書き、パスするまで修正をループ。
  • Git Ops: ブランチ作成、コミット、PR作成、マージまでをCLIだけで完結。
  • Safe Mode: 破壊的なコマンド（rm -rfなど）は必ず人間の承認を要求。

The "Team" Inside New in 2026

Claude Codeはもはや単独ではありません。タスクに応じて軽量なサブエージェント（Haiku/Sonnetクラス）を召喚し、チームとして動きます。

1

Planner (Opus)

全体のタスクを分解し、どのファイルを誰が担当するかを決定する司令塔。

2

Workers (Sonnet)

並列してFrontend、Backend、Databaseのコード修正を同時に進行。

3

Reviewer (Haiku)

生成されたコードのLintチェックや、命名規則の統一を高速にチェック。

> claude task "Implement User Auth" [Planner] Analyzing requirements... [Planner] Plan created: 1. Update DB Schema (users table) 2. Implement API (auth routes) 3. Create Frontend (login form) [Starting Sub-Agents...] [Worker-1] Working on users.sql... Done. [Worker-2] Working on api/auth.ts... Done. [Worker-3] Working on components/Login.vue... Done. [Reviewer] Checking diffs... WARNING: 'password' field in users.sql should be hashed. [Worker-1] Fixing SQL schema... Done. All tasks completed. Ready to merge? [Y/n]

Skills & MCP

What is MCP?

Model Context Protocol (MCP) は、Claude Codeの手足を拡張する標準規格です。MCPサーバーを追加することで、Claudeはローカル環境の外にある世界にもアクセスできるようになります。

Essential Skills

  • Postgres Skill: 本番DBに（読み取り専用で）接続し、実際のデータ分布を確認しながらクエリを最適化。
  • AWS/GCP Skill: インフラ構成を確認したり、デプロイログを直接取得して解析。
  • Slack/Notion Skill: 「仕様書ドキュメント」や「過去の議論」を検索し、コンテキストとして取り込む。

Co-Work: The "Vibe" Style

Claude Codeとの理想的な働き方（Co-work）は、上司と部下の関係ではなく、「ベテラン同士の阿吽の呼吸」です。

❌ Bad Vibe (Micro-management)

「このファイルの5行目を直して」「次はここをこうして」と細かく指示しすぎる。
→ Clause Codeの自律性を殺してしまい、逆に遅くなる。

✅ Good Vibe (Goal-oriented)

「ログイン画面のバリデーションが甘い気がする。全体的に見直して堅牢にしてくれ」
→ 意図を汲み取り、関連するすべての層（UI、API、DB）を横断して提案を持ってくる。
